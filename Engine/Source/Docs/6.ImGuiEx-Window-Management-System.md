# Neverness Engine — ImGuiEx Window 管理系统架构设计

## Context

当前 Neverness Engine Editor 的 UI 管理系统非常基础：
- `PanelManager` 单例持有两个 `List/Dictionary`，逐帧遍历调用 `OnGUI()`
- `EditorMainWindow` 手动创建 DockSpace，手动调用 `SetNextWindowClass` 分配子面板
- 无 Toast 通知、无异步任务 UI、无 Modal 窗口抽象、无 Texture Viewer
- `IPanel`/`IEditorPanel` 接口简陋，缺少生命周期、多实例、窗口 ID 体系

目标：在 `Neverness.Editor.ImGuiEx` 项目中实现一套完整的、工业级的 ImGui Window 管理框架，
类似 Unity EditorWindow + Unreal Slate TabManager + Dear ImGui ToolHost。

## 核心类详细设计

### 1. ImWindow — 窗口抽象基类

**文件**: `Public/Window/ImWindow.cs`

核心职责：封装 `ImGui.Begin/End`，管理窗口生命周期，支持多实例、Docking、焦点跟踪。

```
字段:
  Guid WindowId           — 构造时 Guid.NewGuid()，保证多实例唯一
  string Title            — 窗口标题
  bool _isOpen / _isVisible / _wasFocused / _disposed
  ImGuiWindowFlags Flags
  uint DockId
  ImWindowBehaviorFlags BehaviorFlags

生命周期模板方法 (virtual, 子类覆写):
  OnOpen() / OnClose() / OnFocus() / OnLostFocus() / OnUpdate(float)

渲染模板方法 (子类必须实现):
  abstract OnRender()            — 绘制窗口内容
  virtual OnRenderMenuBar()      — 绘制菜单栏
  virtual bool HasMenuBar()

Render() — 非虚模板方法，框架控制:
  1. if (!visible || !open) return
  2. SetNextWindowDockID(DockId)
  3. SetNextWindowClass(ref windowClass)
  4. Begin(GetImGuiId(), ref open, Flags)  — "Title##GUID_HEX"
  5. 检测 open 变化 → OnClose()
  6. 检测焦点变化 → OnFocus/OnLostFocus
  7. if HasMenuBar → BeginMenuBar → OnRenderMenuBar → EndMenuBar
  8. OnRender()
  9. End()

RequestClose() — _isOpen = false, OnClose()
Dispose() — OnClose()
```

关键设计决策：
- `GetImGuiId()` 返回 `$"{Title}##{WindowId:N}"` — ImGui 标准多实例方案
- `Render()` 是模板方法（非虚），子类只实现 `OnRender()` — 框架保证 Begin/End 配对
- 每个窗口自动创建独立 `ImGuiWindowClass` 用于 Docking 分类

### 2. ImWindowManager — 窗口管理器

**文件**: `Private/Window/ImWindowManager.cs`, 接口 `Public/Window/IImWindowManager.cs`

```
存储:
  Dictionary<Guid, ImWindow> _windows
  Dictionary<Type, Func<ImWindow>> _factories

核心 API:
  OpenWindow<T>(Action<T>? configure)  — 创建并打开窗口
    → 非 MultiInstance 类型复用已有实例
    → MultiInstance 类型每次 new
  CloseWindow(Guid) / CloseAll()

  RenderAllWindows() — 遍历所有窗口调用 Render()
  Update(float)      — 遍历所有窗口调用 OnUpdate()
  RegisterType<T>() / RegisterType(Type) — 注册窗口工厂

事件:
  WindowOpened / WindowClosed / WindowFocused
```

### 5. IModalWindow / ModalWindow — Modal 弹窗

**文件**: `Public/Modal/IModalWindow.cs`, `Public/Modal/ModalWindow.cs`

```
字段:
  string Id, bool IsOpen, bool _shouldOpen
  Func<bool>? ContentRenderer  — 返回 true 表示确认
  Action? OnConfirmed / OnCancelled

API:
  Open()   — 标记下一帧 OpenPopup
  Close()  — CloseCurrentPopup
  Render() — 每帧调用，处理 OpenPopup/BeginPopupModal/EndPopup

用法示例:
  var confirm = new ModalWindow("Delete?");
  confirm.ContentRenderer = () => {
      ImGui.Text("确定删除？");
      if (ImGui.Button("Yes")) return true;
      return false;
  };
  confirm.OnConfirmed = () => DeleteAsset();
  // 帧循环中: confirm.Render();
```

### 6. INotificationService / NotificationService — Toast 通知

**文件**: `Public/Notifications/INotificationService.cs`, `Private/Notifications/NotificationService.cs`

```
NotificationLevel: Info, Success, Warning, Error

NotificationItem (readonly record struct):
  Guid Id, string Title, string Message,
  NotificationLevel Level, DateTime CreatedAt, TimeSpan Duration

NotificationService:
  Show(level, title, message, duration?) → Guid
  Info/Success/Warning/Error — 快捷方法
  Dismiss(Guid) / DismissAll()
  Update(float) — 移动 pending → active, 过期自动删除
  Render() — 委托给 ImNotificationRenderer
```

**ImNotificationRenderer** (`Public/Notifications/ImNotificationRenderer.cs`):
- 纯静态方法，接收 `IReadOnlyList<NotificationItem>` 数据
- 视口右下角定位，`NoDocking | NoInputs | NoNav | NoFocusOnAppearing`
- 淡入 (0.25s) / 淡出 (0.15s) 动画
- 左侧色条标识级别，深色半透明背景
- 使用 `ImGuiWindowFlags.NoDecoration` 无边框窗口 + DrawList 自绘

### 7. UIThreadDispatcher — UI 线程调度器

**文件**: `Public/Tasks/UIThreadDispatcher.cs`

```csharp
sealed class UIThreadDispatcher {
    ConcurrentQueue<Action> _queue;
    void Enqueue(Action action);      // 后台线程调用
    void ProcessQueue();              // 主线程每帧调用
}
```

这是线程安全基础原语。所有 `ImGui.*` 调用必须在主线程。
后台线程通过 `dispatcher.Enqueue(() => notificationService.Info(...))` 安全提交 UI 操作。

### 8. IImUITask / ImUITask — 异步任务 UI

**文件**: `Public/Tasks/IImUITask.cs`, `Public/Tasks/ImUITask.cs`

```
ImTaskState: Running, Completed, Faulted, Cancelled

IImUITask:
  Guid TaskId, string Label, ImTaskState State
  float Progress (0..1, -1=indeterminate), string? StatusText
  CancellationToken CancellationToken
  Task RunAsync(UIThreadDispatcher)

ImUITask (abstract基类):
  子类实现 abstract ExecuteAsync(UIThreadDispatcher, CancellationToken)
  基类处理 CTS、状态转换、异常捕获
  Cancel() — 请求取消
```

**ImUITaskWindow** (`Public/Tasks/ImUITaskWindow.cs`):
- 继承 `ImWindow`，`MultiInstance` 可选
- 持有 `List<IImUITask>` 引用
- 渲染所有活跃任务：进度条 + 状态文本 + 取消按钮

### 9. ImDockLayoutService — 停靠布局服务

**文件**: `Public/Docking/IImDockLayoutService.cs`, `Private/Docking/ImDockLayoutService.cs`

```
DockZone (readonly record struct):
  string Name, ImGuiDir Direction, float Ratio

IImDockLayoutService:
  uint MainDockSpaceId
  void Initialize(uint dockspaceId)
  void BuildDefaultLayout(IReadOnlyList<DockZone> zones)
  void DockWindow(string windowName, string zoneName)
  void SaveLayout() / RestoreLayout()
  bool IsLayoutBuilt

DockLayoutBuilder (内部实现):
  使用 DockBuilderAddNode / DockBuilderSplitNode / DockBuilderSetNodeSize / DockBuilderFinish
  构建默认布局

集成方式:
  现有 DockingLayout (Framework) 的 zone 数据 → 转换为 DockZone 列表 → 传入 BuildDefaultLayout
  布局持久化使用 ImGui.SaveIniSettingsToDisk/LoadIniSettingsFromDisk
```

### 10. ImTextureViewerWindow — 纹理查看器

**文件**: `Public/Viewer/ImTextureViewerWindow.cs`

```
继承 ImWindow，BehaviorFlags = MultiInstance | Default
Flags = ImGuiWindowFlags.MenuBar

功能:
  SetTexture(IntPtr textureId, Vector2 size, int mipCount)
  菜单栏: View (Fit/Checkerboard/RGBA toggle) + Mip (mip level 选择)
  渲染: Checkerboard 背景 + 纹理缩放显示
  鼠标交互: 滚轮缩放 (0.1x~50x), 中键拖拽平移
  Fit-to-Window 自动缩放模式
  状态栏: Zoom + Size 信息

注意: RGBA 通道切换需要 GPU shader/texture swizzle，
当前版本仅预留 UI 接口，实际通道过滤需底层图形 API 支持。
```

## 编辑器集成

### IEditorFeature 注册

在 `Neverness.Editor.Core` 中新建 `ImGuiWindowFeature`:

```csharp
sealed class ImGuiWindowFeature : IEditorFeature {
    FeatureId = "com.neverness.imgui-windows"
    Dependencies = ["com.neverness.framework"]

    Initialize(context):
        dispatcher = new UIThreadDispatcher()
        windowManager = new ImWindowManager()
        notificationService = new NotificationService()
        dockLayoutService = new ImDockLayoutService()

        context.RegisterService<IImWindowManager>(windowManager)
        context.RegisterService<INotificationService>(notificationService)
        context.RegisterService<IImDockLayoutService>(dockLayoutService)
        context.RegisterService(dispatcher)

    Shutdown(context):
        windowManager.CloseAll()
        notificationService.DismissAll()
```

### 插件注册示例

```csharp
// 插件的 IEditorFeature.Initialize():
void Initialize(IEditorContext ctx) {
    var windows = ctx.GetService<IImWindowManager>();
    windows.RegisterType<MyPluginToolWindow>();
    windows.OpenWindow<MyPluginToolWindow>();

    var notifications = ctx.GetService<INotificationService>();
    notifications.Success("Plugin", "MyPlugin 已加载");

    // 后台任务:
    var dispatcher = ctx.GetService<UIThreadDispatcher>();
    Task.Run(async () => {
        var task = new MyImportTask("导入资源...");
        await task.RunAsync(dispatcher);
    });
}
```

## 关键架构决策

| 决策 | 理由 |
|------|------|
| Guid 作为窗口 ID | 天然多实例支持，无冲突 |
| `##GUID` 后缀 ImGui ID | ImGui 标准多窗口方案 |
| Render() 非虚模板方法 | 框架控制 Begin/End 配对，子类只管 OnRender |
| ImWindowBehaviorFlags 位标志 | 紧凑、可扩展，一个字段定义窗口全部行为 |
| 通知渲染器分离为静态类 | 纯数据→渲染，可测试，无服务依赖 |
| ConcurrentQueue<Action> 调度器 | 最简正确方案，无锁 |
| 接口优先设计 | 支持服务定位器注册，支持 Mock 测试 |
| 布局 INI 持久化复用 ImGui 内建机制 | 不重复造轮子 |
| Notification 在最后渲染 | 保证在所有窗口最上层，不被遮挡 |

### 与原计划的偏差

1. **DockBuilder API**: 原计划使用 `ImGui.DockBuilder*`，实际 API 在 `ImGuiP` 类中 (Internals)。
   修改为使用 `ImGuiP.DockBuilderAddNode` / `ImGuiP.DockBuilderDockWindow` 等。

2. **ImGuiDockNodeFlags.DockSpace**: 此枚举值不存在于当前版本的 Hexa.NET.ImGui。
   改用 `ImGuiDockNodeFlags.None`。

3. **ImUITaskWindow 图标**: 原计划使用 FontAwesome5Pro 图标，但 ImGuiEx 项目不引用 Framework。
   改用 ASCII 标记 `[~]` / `[v]` / `[!]` / `[x]`。

4. **NotificationService 线程安全**: 原计划中 Show() 是线程安全的但未显式标注。
   实现中添加了 `lock` 以确保后台线程可安全调用。
