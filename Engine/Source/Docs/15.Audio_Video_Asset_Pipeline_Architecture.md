| TypeId | 名称 | 状态 |
|---|---|---|

| 4 | Material | 已实现 |

### 1.3 TypeInfo 结构体

**NNAudioTypeInfo**（已存在，24 字节，`#pragma pack(push, 8)`）：

```c
typedef struct NNAudioTypeInfo {
    uint32_t sampleRate;      // 偏移 0,  4B — 采样率（44100/48000）
    uint32_t channels;        // 偏移 4,  4B — 声道数（1=mono, 2=stereo）
    uint64_t sampleCount;     // 偏移 8,  8B — 总采样数
    uint32_t format;          // 偏移 16, 4B — NNAudioCompressionFormat 枚举
    uint32_t flags;           // 偏移 20, 4B — bit0=streaming
} NNAudioTypeInfo;
// sizeof = 24, alignof = 8
```

**NNVideoTypeInfo**（新增，48 字节，`#pragma pack(push, 8)`）：

```c
typedef struct NNVideoTypeInfo {
    uint32_t width;           // 偏移 0,  4B — 视频宽度
    uint32_t height;          // 偏移 4,  4B — 视频高度
    uint32_t fpsNum;          // 偏移 8,  4B — 帧率分子
    uint32_t fpsDen;          // 偏移 12, 4B — 帧率分母
    uint64_t frameCount;      // 偏移 16, 8B — 总帧数
    double   duration;        // 偏移 24, 8B — 时长（秒）
    uint32_t codecId;         // 偏移 32, 4B — NNVideoCodec 枚举
    uint32_t flags;           // 偏移 36, 4B — 见 NNVideoFlags
    uint32_t audioSampleRate; // 偏移 40, 4B — 音轨采样率（0=无音轨）
    uint32_t audioChannels;   // 偏移 44, 4B — 音轨声道数
} NNVideoTypeInfo;
// sizeof = 48, alignof = 8
```

## 2. Asset Importer 架构

### 2.1 设计原则

1. **FFmpeg 永远不暴露到 C# 层** — C# 只调用 C++ 桥接 API
2. **遵循现有 `IAssetImporter` 模式** — `[AssetImporter]` 特性 + `Import()` 方法
3. **Cook 操作在 C++ 完成** — C# 只做编排和 UI
4. **增量导入** — SHA-256 源文件哈希检测变化

### 2.2 C# Importer 层

#### AudioAssetImporter

```csharp
// Neverness.Editor.MediaImporter/Public/AudioAssetImporter.cs
[AssetImporter(".wav", ".mp3", ".ogg", ".flac", Priority = 100)]
public class AudioAssetImporter : IAssetImporter, ISettingsAwareImporter
{
    public string DisplayName => "Audio Clip";
    public string[] SupportedExtensions => new[] { ".wav", ".mp3", ".ogg", ".flac" };

    public ImportResult Import(AssetImportContext context)
    {
        // 1. 读取 .meta 中的 AudioImportSettings
        var settings = context.GetSettings<AudioImportSettings>();

        // 2. 调用 C++ API 探测媒体信息
        var probe = MediaImportBridge.ProbeFile(context.SourcePath);

        // 3. 调用 C++ API 提取 PCM 数据
        byte[] pcmData = MediaImportBridge.ExtractPCM(
            context.SourcePath,
            settings.ForceToMono ? 1 : probe.Channels,
            settings.TargetSampleRate);

        // 4. 调用 C++ API 生成音频波形缩略图
        byte[] thumbnail = MediaImportBridge.GenerateAudioThumbnail(
            context.SourcePath, 256, 64);

        // 5. 可选：调用 C++ API 生成 Seek 表
        byte[] seekTable = settings.Streaming
            ? MediaImportBridge.BuildSeekTable(context.SourcePath)
            : null;

        // 6. 构建 ImportResult
        var result = new ImportResult
        {
            AssetGuid = context.Guid,
            TypeId = AssetTypeId.AudioClip,
            ThumbnailData = thumbnail,
        };

        // NNAudioTypeInfo (TYPE_INFO blob)
        result.TypeInfo = AudioTypeInfoHelper.Serialize(
            probe.SampleRate, probe.Channels,
            CalculateSampleCount(pcmData, probe),
            settings.CompressionFormat,
            settings.Streaming);

        // AUDIO_PCM blob
        result.Blobs.Add(new ImportedBlob
        {
            BlobType = BlobType.AudioPcm,
            Data = pcmData
        });

        // AUDIO_SEEK blob（仅流式）
        if (seekTable != null)
        {
            result.Blobs.Add(new ImportedBlob
            {
                BlobType = BlobType.AudioSeek,
                Data = seekTable
            });
        }

        return result;
    }

    public object GetDefaultSettings() => AudioImportSettings.Default;
    public bool ValidateSettings(object settings, out string error) { ... }
}
```

#### VideoAssetImporter

```csharp
// Neverness.Editor.MediaImporter/Public/VideoAssetImporter.cs
[AssetImporter(".mp4", ".webm", ".mov", ".mkv", ".avi", Priority = 100)]
public class VideoAssetImporter : IAssetImporter, ISettingsAwareImporter
{
    public string DisplayName => "Video Clip";
    public string[] SupportedExtensions => new[] { ".mp4", ".webm", ".mov", ".mkv", ".avi" };

    public ImportResult Import(AssetImportContext context)
    {
        var settings = context.GetSettings<VideoImportSettings>();

        // 1. C++ FFmpeg 探测
        var probe = MediaImportBridge.ProbeFile(context.SourcePath);

        // 2. C++ FFmpeg 解码首帧 → 缩略图
        byte[] thumbnail = MediaImportBridge.DecodeThumbnail(
            context.SourcePath, settings.MaxResolution);

        // 3. 流式模式：保留原始编码数据
        //    非流式模式：全量解码为 RGBA 帧序列
        byte[] encodedData = null;
        byte[][] decodedFrames = null;

        if (settings.Streaming)
        {
            // 将原始媒体文件内容作为 DATA blob 内嵌
            encodedData = File.ReadAllBytes(context.SourcePath);
        }
        else
        {
            // C++ 全量解码
            decodedFrames = MediaImportBridge.DecodeAllFrames(
                context.SourcePath, settings.MaxResolution);
        }

        // 4. 提取音轨（可选）
        byte[] audioPcm = null;
        if (settings.IncludeAudioTrack && probe.HasAudio)
        {
            audioPcm = MediaImportBridge.ExtractPCM(context.SourcePath, 2, 44100);
        }

        // 5. 构建结果
        var result = new ImportResult
        {
            AssetGuid = context.Guid,
            TypeId = AssetTypeId.VideoClip,
            ThumbnailData = thumbnail,
        };

        // NNVideoTypeInfo (TYPE_INFO blob)
        result.TypeInfo = VideoTypeInfoHelper.Serialize(probe, settings);

        // DATA blob（流式 = 原始编码；非流式 = 首帧 RGBA）
        if (settings.Streaming)
        {
            result.Blobs.Add(new ImportedBlob
            {
                BlobType = BlobType.Data,
                Data = encodedData
            });
        }
        else
        {
            for (int i = 0; i < decodedFrames.Length; i++)
            {
                result.Blobs.Add(new ImportedBlob
                {
                    BlobType = BlobType.VideoFrame,
                    Data = decodedFrames[i]
                });
            }
        }

        // VIDEO_SEEK blob（流式模式）
        if (settings.Streaming)
        {
            byte[] seekTable = MediaImportBridge.BuildVideoSeekTable(context.SourcePath);
            result.Blobs.Add(new ImportedBlob
            {
                BlobType = BlobType.VideoSeek,
                Data = seekTable
            });
        }

        // AUDIO_PCM blob（提取的音轨）
        if (audioPcm != null)
        {
            result.Blobs.Add(new ImportedBlob
            {
                BlobType = BlobType.AudioPcm,
                Data = audioPcm
            });
        }

        return result;
    }

    public object GetDefaultSettings() => VideoImportSettings.Default;
    public bool ValidateSettings(object settings, out string error) { ... }
}
```

## 3. 音频 Asset 格式

### 3.1 Source Asset（源文件）

原始音频文件（.wav/.mp3/.ogg/.flac）存放在 `Assets/` 目录。Editor 通过 `AssetWatcher` 检测变更，触发导入。

```
Assets/
  BGM/
    battle_theme.mp3         ← 源文件
    battle_theme.mp3.meta    ← 导入配置（YAML）
```

### 3.2 Cooked Runtime Asset（.nnasset）

导入后生成的 `.nnasset` 二进制文件，存储在 `Library/Imported/{hex[0:2]}/{guid32}.nnasset`。

```
┌──────────────────────────────────────────────────────┐
│ NNAssetHeader (96 bytes)                              │
│   magic    = 'NNAS' (0x4E4E4153)                      │
│   version  = 2                                         │
│   assetGuid = {GUID}                                   │
│   typeId   = NN_TYPE_ID_AUDIO_CLIP (3)                 │
│   flags    = NN_ASSET_FLAG_STREAMING (如果流式)         │
│   blobCount = 2 (PCM + Seek) 或 1 (仅 PCM)             │
├──────────────────────────────────────────────────────┤
│ Blob Descriptor 0: NN_BLOB_TYPE_TYPE_INFO (9)         │
│   → NNAudioTypeInfo (24 bytes)                         │
├──────────────────────────────────────────────────────┤
│ Blob Descriptor 1: NN_BLOB_TYPE_AUDIO_PCM (5)         │
│   → PCM16/Float32/Opus/Vorbis 编码的音频数据            │
├──────────────────────────────────────────────────────┤
│ Blob Descriptor 2: NN_BLOB_TYPE_AUDIO_SEEK (6) [可选]  │
│   → Seek 表条目数组                                     │
├──────────────────────────────────────────────────────┤
│ Blob Descriptor 3: NN_BLOB_TYPE_THUMBNAIL (4) [可选]   │
│   → 音频波形可视化 PNG/RGA 图像                         │
├──────────────────────────────────────────────────────┤
│ Payload（64-byte aligned）                             │
│   [TypeInfo blob: 24 bytes]                            │
│   [PCM blob: N bytes]                                  │
│   [Seek table blob: M bytes]                           │
│   [Thumbnail blob: K bytes]                            │
└──────────────────────────────────────────────────────┘
```

### 3.3 全量加载 vs 流式加载

| 模式 | flags | PCM 格式 | Seek 表 | 适用场景 |
|---|---|---|---|---|
| 全量加载 | 0 | PCM16 | 无 | SFX、UI 音效（< 5MB） |
| 流式加载 | `NN_AUDIO_FLAG_STREAMING` | Opus/Vorbis | 有 | 背景音乐、长音频（> 5MB） |

**全量加载：** `NNAssetManager::LoadAssetSync()` 一次性读取 PCM blob 到内存，`AudioPlayer` 直接从内存播放。

**流式加载：** `NNAssetManager` 只读取 header + TypeInfo + Seek 表。实际 PCM 数据按需通过 Seek 表偏移从 `.nnasset` 文件中分块读取，由 `FAudioDecoder` 在独立线程中解码。

### 3.4 NNAudioTypeInfo 字段说明

```c
typedef struct NNAudioTypeInfo {
    uint32_t sampleRate;      // 44100 或 48000
    uint32_t channels;        // 1=mono, 2=stereo
    uint64_t sampleCount;     // 总采样数 → Duration = sampleCount / sampleRate
    uint32_t format;          // NNAudioCompressionFormat 枚举值
    uint32_t flags;           // NN_AUDIO_FLAG_* 位标志
} NNAudioTypeInfo;
```

### 4.2 Cooked Runtime Asset

#### 流式模式（大视频、长视频）

```
┌──────────────────────────────────────────────────────┐
│ NNAssetHeader (96 bytes)                              │
│   typeId   = NN_TYPE_ID_VIDEO_CLIP (10)               │
│   flags    = NN_ASSET_FLAG_STREAMING                   │
│   blobCount = 3                                        │
├──────────────────────────────────────────────────────┤
│ Blob 0: NN_BLOB_TYPE_TYPE_INFO (9)                    │
│   → NNVideoTypeInfo (48 bytes)                         │
├──────────────────────────────────────────────────────┤
│ Blob 1: NN_BLOB_TYPE_DATA (0)                         │
│   → 原始编码视频数据（MP4/WebM 容器）                    │
├──────────────────────────────────────────────────────┤
│ Blob 2: NN_BLOB_TYPE_THUMBNAIL (4)                    │
│   → 首帧 RGBA 缩略图                                   │
├──────────────────────────────────────────────────────┤
│ Blob 3: NN_BLOB_TYPE_VIDEO_SEEK (11) [可选]            │
│   → 视频 Seek 表（关键帧偏移索引）                       │
├──────────────────────────────────────────────────────┤
│ Payload                                                 │
│   [TypeInfo: 48 bytes]                                 │
│   [Encoded data: N bytes — 原始 MP4/WebM]               │
│   [Thumbnail: W*H*4 bytes]                             │
│   [Seek table: M bytes]                                │
└──────────────────────────────────────────────────────┘
```

#### 非流式模式（短视频、表情动画、WebM Alpha）

```
┌──────────────────────────────────────────────────────┐
│ NNAssetHeader (96 bytes)                              │
│   typeId   = NN_TYPE_ID_VIDEO_CLIP (10)               │
│   flags    = 0 (非流式)                                │
│   blobCount = 1 + FrameCount                          │
├──────────────────────────────────────────────────────┤
│ Blob 0: NN_BLOB_TYPE_TYPE_INFO (9)                    │
│   → NNVideoTypeInfo (48 bytes)                         │
├──────────────────────────────────────────────────────┤
│ Blob 1..N: NN_BLOB_TYPE_VIDEO_FRAME (10)              │
│   → 每帧一个 blob，RGBA8 像素数据                      │
│   → 帧索引 = blob index - 1                            │
├──────────────────────────────────────────────────────┤
│ Blob N+1: NN_BLOB_TYPE_AUDIO_PCM (5) [可选]            │
│   → 内嵌 PCM16 音频数据                                 │
├──────────────────────────────────────────────────────┤
│ Payload                                                 │
│   [TypeInfo: 48 bytes]                                 │
│   [Frame 0: W*H*4 bytes]                               │
│   [Frame 1: W*H*4 bytes]                               │
│   ...                                                   │
│   [Frame N-1: W*H*4 bytes]                             │
│   [Audio PCM: A bytes]                                 │
└──────────────────────────────────────────────────────┘
```

### 4.3 NNVideoTypeInfo 字段说明

```c
typedef struct NNVideoTypeInfo {
    uint32_t width;           // 视频像素宽度
    uint32_t height;          // 视频像素高度
    uint32_t fpsNum;          // 帧率分子（如 30000）
    uint32_t fpsDen;          // 帧率分母（如 1001）→ 实际 FPS = fpsNum/fpsDen = 29.97
    uint64_t frameCount;      // 总帧数
    double   duration;        // 总时长（秒）
    uint32_t codecId;         // NNVideoCodec 枚举
    uint32_t flags;           // NNVideoFlags 位标志
    uint32_t audioSampleRate; // 音轨采样率（0=无音轨）
    uint32_t audioChannels;   // 音轨声道数
} NNVideoTypeInfo;
```

### 4.4 流式 vs 非流式对比

| 特性 | 流式 | 非流式 |
|---|---|---|
| .nnasset 大小 | 小（原始编码，通常 1-100MB/分钟） | 大（全量 RGBA，150MB/秒 @1080p） |
| 加载速度 | 快（只读 header + seek 表） | 慢（需读取所有帧） |
| 内存占用 | 低（按需解码 1-3 帧缓冲） | 高（全量帧在内存中） |
| Seek 速度 | 中（靠 Seek 表定位关键帧） | 快（直接索引 blob） |
| 适用场景 | 长视频、过场动画、BGM 视频 | 短表情、Live2D 动画、WebM Alpha |
| FFmpeg 依赖 | 运行时必须 | 运行时不需要 |

### 4.5 Alpha 通道支持

WebM (VP8/VP9) 视频可含 Alpha 通道，用于 Live2D、动态表情等。

- `NNVideoTypeInfo.flags` 包含 `NN_VIDEO_FLAG_ALPHA_CHANNEL`
- 解码输出为 RGBA（而非 RGB）
- `NNVideoTexture` 创建时使用 `GL_RGBA` 而非 `GL_RGB`
- `Renderer2D` 的 SpriteShader 需启用 Alpha Blend

### 5.1 整体流程

### 6.6 NNVideoTexture（关键：弥合断层）

这是当前代码库中缺失的关键组件。`FVideoPlayer::GetFrameData()` 返回 `uint8_t*` RGBA 但 GPU 上传代码被注释掉。`NNVideoTexture` 弥合这一断层。

```cpp
// NNRuntimeMedia/Include/NNVideoTexture.h

class NNVideoTexture
{
public:
    ~NNVideoTexture() { Release(); }

    // 初始化 GPU 纹理（首次调用）
    void Initialize(uint32_t width, uint32_t height);

    // 每帧更新：将 RGBA 像素上传到 GPU
    // 这是弥合 FVideoPlayer ↔ NNRenderAssetManager 断层的核心方法
    void UpdateFrame(const uint8_t* rgbaPixels, uint32_t width, uint32_t height);

    // 获取 GPU 纹理标识
    uint64_t GetCacheKey() const { return m_CacheKey; }
    uint64_t GetImGuiHandle() const;
    uint32_t GetGLTextureId() const;

    // 释放 GPU 资源
    void Release();

private:
    uint64_t m_CacheKey = 0;      // NNRenderAssetManager 的缓存 key
    uint32_t m_Width = 0;
    uint32_t m_Height = 0;
};
```

**核心实现（弥合断层）：**

```cpp
// NNRuntimeMedia/Source/NNVideoTexture.cpp

void NNVideoTexture::Initialize(uint32_t width, uint32_t height)
{
    m_Width = width;
    m_Height = height;

    // 创建 1x1 占位纹理，后续帧通过 UpdateFrame 热更新
    uint8_t blackPixel[4] = {0, 0, 0, 255};
    m_CacheKey = NNRenderAssetManager::Get().CreateTextureFromPixels(
        blackPixel, 1, 1, 4, false);
}

void NNVideoTexture::UpdateFrame(const uint8_t* rgbaPixels, uint32_t width, uint32_t height)
{
    if (!rgbaPixels) return;

    if (width != m_Width || height != m_Height)
    {
        // 分辨率变化：重建纹理
        Release();
        Initialize(width, height);
    }

    // ★ 关键调用：热更新 GPU 纹理像素 ★
    // 这填补了 FVideoPlayer::GetFrameData() → GPU 之间的断层
    NNRenderAssetManager::Get().UpdateTexturePixels(
        m_CacheKey, rgbaPixels, width * height * 4);
}

uint64_t NNVideoTexture::GetImGuiHandle() const
{
    return NNRenderAssetManager::Get().GetImGuiTextureHandle(m_CacheKey);
}
```

```cpp
// NNRuntimeRenderAssets/Private/NNRenderAssetManager.cpp

bool NNRenderAssetManager::UpdateTexturePixels(uint64_t cacheKey, const uint8_t* pixels, size_t pixelSize)
{
    std::lock_guard<std::mutex> lock(m_Mutex);

    auto it = m_EntryCache.find(cacheKey);
    if (it == m_EntryCache.end()) return false;

    auto* entry = it->second.get();
    if (!entry || !entry->OwnedTexture) return false;

    // 通过 type-erased shared_ptr<void> 获取 OpenGL texture
    // 需要存储一个 update 回调而非仅 deleter
    if (entry->UpdateCallback)
    {
        entry->UpdateCallback(pixels, pixelSize);
        return true;
    }

    return false;
}
```

### 6.8 A/V 同步

现有 `FVideoPlayer::ProcessVideoUpdate()` 已实现 PTS-based timing（使用系统时钟）。`NNVideoPlayer` 包装它并增加：

## 8. Thumbnail 系统

### 8.1 视频缩略图

**生成时机：** Import Phase 3（`VideoAssetImporter.Import()`）

**生成方式：**
1. C# 调用 `MediaImportBridge.DecodeThumbnail(sourcePath, maxWidth: 256)`
2. C++ 内部使用 FFmpeg 解码第一个关键帧
3. `sws_scale` 缩放到目标尺寸，输出 RGBA
4. 返回 `byte[]` RGBA 数据
5. C# 将其写入 `THUMBNAIL` blob 并存入 `.nnasset`
6. 同时写入 `Library/Thumbnails/{guid}.png`（Content Browser 显示用）

**显示方式：**
- Content Browser：从 `Library/Thumbnails/{guid}.png` 加载 `ImTextureID`
- Inspector：从 `.nnasset` 的 `THUMBNAIL` blob 加载 → `NNRenderAssetManager::LoadTextureFromBlob()` → ImGui handle

### 8.2 音频波形图

**生成时机：** Import Phase 3（`AudioAssetImporter.Import()`）

**生成方式：**
1. C# 调用 `MediaImportBridge.GenerateAudioThumbnail(sourcePath, width: 256, height: 64)`
2. C++ 内部：
   - 解码 PCM 数据（或读取 WAV 头直接获取）
   - 按 `width` 个采样点分桶
   - 每个桶计算 min/max 振幅
   - 绘制波形到 RGBA 图像（背景透明，波形白色/浅色）
3. 返回 RGBA 像素数据

**存储：** 波形图作为 `NN_BLOB_TYPE_THUMBNAIL` blob 内嵌在 `.nnasset` 中。

### 8.3 缓存策略

- 缩略图数据内嵌在 `.nnasset` 文件中，无需独立缓存
- `Library/Thumbnails/{guid}.png` 用于 Content Browser 快速预览（与现有纹理缩略图一致）
- 源文件变更时，ImportPipeline 的 Phase 7 自动重新生成

## 11. C#/C++ 边界

### 11.1 必须 Native（C++）的操作

| 操作 | 原因 |
|---|---|
| FFmpeg 所有调用 | FFmpeg 是 C 库，不应暴露到 Managed 环境 |
| PCM 解码（`swr_convert`） | 音频重采样是 CPU 密集操作 |
| 视频帧解码（`avcodec` + `sws_scale`） | 涉及硬件加速，需要直接内存访问 |
| SDL3 音频回调 | 实时音频线程，延迟敏感 |
| GPU 纹理上传（`glTextureSubImage2D`） | OpenGL 调用必须在 GL 上下文线程 |
| Seek 表构建 | 需要 FFmpeg 的 `av_seek_frame` |
| Opus/Vorbis 编码 | 编码器是 C 库 |
| 流式解码线程 | 多线程 + Ring Buffer，不适合 P/Invoke 频繁跨越 |

### 11.2 可以 C# 的操作

| 操作 | 原因 |
|---|---|
| Import Settings UI | ImGui Inspector，纯 UI 逻辑 |
| 资产发现（Content Browser 扫描） | 文件系统遍历，无性能要求 |
| Cook Manifest 构建 | 数据组装，无计算密集操作 |
| .meta 文件读写 | YAML 解析，已有 `MetaFileManager` |
| 进度报告 UI | Editor UI 更新 |
| Importer 注册/发现 | 反射扫描，已有 `ImporterRegistry` |

### 11.3 禁止 C# 的操作

| 操作 | 原因 |
|---|---|
| FFmpeg | 绝不暴露 |
| 大块像素数据 P/Invoke | 1080p RGBA = 8MB/帧，Marshal 开销巨大 |
| 音频回调线程 | 实时性要求，GC 暂停会导致爆音 |
| 视频解码线程 | 与 GL 上下文交互，线程安全问题 |
| 纹理像素数据 | 应通过 C++ 直接操作 GPU 内存 |

## 12. Runtime Streaming 策略

### 12.1 分级策略

| 使用场景 | 加载策略 | 内存模式 | 解码方式 | 理由 |
|---|---|---|---|---|
| SFX / UI 音效 | 全量加载 | PCM 在内存 | 无需解码 | 小文件（< 5MB），需即时播放 |
| 背景音乐 | Ring Buffer 流式 | 编码数据 + 2MB Buffer | 实时 Opus 解码 | 大文件（> 50MB），连续播放 |
| 过场动画 | 异步解码流式 | 1-3 帧 RGBA 缓冲 | FFmpeg 三线程 | A/V 同步关键，需 seek |
| 开放世界环境音 | 距离优先流式 | 动态 | 按需解码 | 内存预算驱动，远距离降质 |
| Visual Novel 对话 | 预加载 + Seek 表 | Seek 表 + 关键帧缓存 | 按需解码 | 频繁 seek，快速响应 |
| Live2D / WebM Alpha | 全量解码 | 全量 RGBA 帧 | 逐帧精确 | Alpha 通道需逐帧同步 |
| 短视频表情（< 10s） | 全量预加载 | 全量 RGBA 帧 | 无需运行时解码 | 文件小，无 FFmpeg 运行时依赖 |

### 12.2 Ring Buffer 流式（音频）

复用 `NNMediaCore::AudioRingBuffer`（2MB 默认容量）：

```
.nnasset 文件（Opus 编码）
    │
    ▼
IO Thread ──── 从 VFS 读取编码数据块
    │
    ▼
FAudioDecoder ──── swr_convert → PCM16
    │
    ▼
AudioRingBuffer (2MB) ──── 生产者/消费者模式
    │
    ▼
SDL3 Audio Callback ──── 从 RingBuffer 读取 → 扬声器
```

### 12.3 异步视频解码

复用 `NNMediaCore::FVideoDecoder` 三线程架构：

```
DemuxThread ──── 读取数据包 → 分发到 VideoPacketQueue / AudioPacketQueue
    │
    ├── AudioDecodeThread ──── 解码音频帧 → AudioRingBuffer → SDL3
    │
    └── VideoDecodeThread ──── 解码视频帧 → RGBA → OnVideoDataUpdate 回调
                                                                    │
                                                                    ▼
                                                            NNVideoTexture::UpdateFrame()
                                                                    │
                                                                    ▼
                                                            NNRenderAssetManager::UpdateTexturePixels()
```

void NNVideoPlayerSystem::OnUpdate(float dt)
{
    auto view = registry.view<NNSpriteRendererComponent, NNVideoPlayerComponent>();

    for (auto entity : view)
    {
        auto& sprite = view.get<NNSpriteRendererComponent>(entity);
        auto& player = view.get<NNVideoPlayerComponent>(entity);

        if (player.VideoClipAsset.low == 0) continue;

        // 更新播放器（解码 + A/V 同步）
        auto* videoPlayer = GetOrCreatePlayer(player);
        videoPlayer->Update(dt);

        // 将视频纹理 ID 写入 SpriteRenderer
        auto* videoTexture = videoPlayer->GetVideoTexture();
        if (videoTexture && videoTexture->IsValid())
        {
            sprite.TextureRuntimeId = static_cast<uint32_t>(
                videoTexture->GetGLTextureId());
        }
    }
}

    uint32_t RuntimePlayerId = 0;    // 偏移 16,  4B — 运行时播放器句柄（transient）
    uint32_t VideoTextureId = 0;     // 偏移 20,  4B — GPU 纹理 ID（transient）
    float    Volume = 1.0f;          // 偏移 24,  4B — 音量
    uint32_t Flags = 0;              // 偏移 28,  4B — NNVideoPlayerFlags
    NNGuid   TargetSprite{};         // 偏移 32, 16B — 可选：渲染到此 SpriteRenderer
    uint32_t _reserved0 = 0;         // 偏移 48,  4B — 保留
    uint32_t _reserved1 = 0;         // 偏移 52,  4B — 保留
};
// sizeof = 56, alignof = 4

// 视频播放器标志位
#define NN_VIDEO_PLAYER_PLAY_ON_AWAKE   (1u << 0)   // 场景启动时自动播放
#define NN_VIDEO_PLAYER_LOOP            (1u << 1)   // 循环播放
#define NN_VIDEO_PLAYER_RENDER_TO_SPRITE (1u << 2)  // 渲染到 SpriteRenderer
#define NN_VIDEO_PLAYER_MUTE            (1u << 3)   // 静音
#define NN_VIDEO_PLAYER_PAUSED          (1u << 4)   // 暂停状态

### 14.6 Scene 序列化

组件通过现有的 `NNSceneSerializer`（二进制）和 `NNJsonSceneSerializer`（JSON）自动序列化。`RuntimePlayerId` 和 `VideoTextureId` 标记为 transient，不参与序列化。

**关键：** `NNGuid` 字段（`AudioClipAsset`、`VideoClipAsset`、`TargetSprite`）使用 `NN_FIELD(..., Guid)` 类型标签，由 `NNSceneSerializer v3` 使用完整 128 位序列化。

### 14.7 空间音频（Phase 2 预留）

`NNAudioSourceComponent` 的 `MinDistance` / `MaxDistance` / `NN_AUDIO_SOURCE_SPATIAL` 标志为 3D 空间音频预留。

Phase 2 实现：
1. `NNAudioSystem` 根据 `NNTransformComponent.Position` 和监听器位置计算衰减
2. SDL3 音频回调中应用距离衰减和立体声声像
3. 支持多普勒效应（可选）

### 15.3 六条架构不变量

> **不变量 1：FFmpeg 不离开 NNMediaCore**
>
> 所有 FFmpeg 类型（`AVFormatContext`、`AVCodecContext`、`AVFrame` 等）仅在 `NNMediaCore` 内部使用。上层通过 `IAudioDecoder` / `IVideoDecoder` 接口或 `MediaImportAPI` C ABI 访问。违反此规则将导致模块耦合和跨 DLL ABI 崩溃。

> **不变量 2：Cook 是 C++ 操作**
>
> C# Editor 编排导入流程（settings 读取、进度 UI、.nnasset 编译），但所有媒体解码/编码操作在 C++ 完成。C# 通过 `NNMediaProbeResult` 等 blittable 结构体获取元信息，绝不直接操作像素或 PCM 数据。

> **不变量 3：ECS 组件是纯 POD 数据**
>
> `NNAudioSourceComponent` 和 `NNVideoPlayerComponent` 必须满足 `trivially_copyable` + `standard_layout`。不包含 `std::string`、`std::vector`、指针或虚函数。所有行为在 System 中实现。RuntimePlayerId / VideoTextureId 是 transient 字段，不序列化。

> **不变量 4：视频纹理必须走 NNRenderAssetManager**
>
> 视频帧的 GPU 上传必须通过 `NNRenderAssetManager::UpdateTexturePixels()`，不得直接调用 OpenGL。这确保纹理生命周期由缓存管理器统一控制，支持 LRU 驱逐和 ImGui handle 获取。

> **不变量 5：所有跨 C#/C++ 边界的结构体必须 blittable**
>
> `NNMediaProbeResult`、`NNAudioTypeInfo`、`NNVideoTypeInfo`、`NNAudioSourceComponentData`、`NNVideoPlayerComponentData` 等必须使用 `[StructLayout(LayoutKind.Sequential)]` + `#pragma pack(push, 8)`。仅使用固定宽度类型（`uint32_t` / `uint64_t` / `double`），不含指针或引用。

> **不变量 6：流式与全量加载是同一资产格式的两种消费方式**
>
> 同一个 `.nnasset` 文件既可以全量加载（读取所有 blob 到内存），也可以流式加载（只读 header + seek 表，按需解码）。两种模式的区别由 `NNAssetHeader.flags` 中的 `NN_ASSET_FLAG_STREAMING` 位控制，而不是不同的文件格式。
