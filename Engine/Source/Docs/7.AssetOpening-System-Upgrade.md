## 纹理加载管线

```
ContentBrowserPanel 双击纹理
  → AssetOpenService.OpenAsync(virtualPath)
    → EditorAssetDatabase.TryGetGuid(virtualPath) → GUID
    → EditorAssetDatabase.GetTypeId(guid) → AssetTypeId.Texture2D
    → AssetOpenerRegistry.GetOpener(typeId) → TextureAssetOpener
    → TextureAssetOpener.OpenAsync(context)
      1. AssetEditorManager.TryGetWindowId(context.Guid)
         → 有：FocusWindow，返回
         → 无：继续
      2. Task.Run → LoadOrImportAsset(context)
         a. AssetHandleExtensions.LoadSync(guid, typeId)
            → 成功：返回 AssetHandle
            → 失败：继续 b
         b. VFS.GetAbsolutePath(virtualPath) → 源文件路径
         c. ImportPipeline.Import(sourcePath) → 导入为 .nnasset
         d. 再次 LoadSync(guid, typeId) → AssetHandle
      3. TextureInterop.LoadTextureFromAsset(assetHandle.Value)
         → ulong textureKey（GPU 缓存 key）
         → C++ 内部：读 blob[0] → NNTextureSourceAsset → OpenGL Texture2D
      4. TextureInterop.GetImGuiTextureHandle(textureKey)
         → ulong imGuiHandle
      5. TextureInterop.GetTextureSize(textureKey)
         → (width, height)
      6. ImWindowManager.OpenWindow<ImTextureViewerWindow>(viewer => {
             viewer.AssetGuidHex = guid.ToHexString();
             viewer.AssetName = virtualPath.FileNameWithoutExtension;
             viewer.SetTexture(imGuiHandle, size);
         })
      7. AssetEditorManager.Register(guid, viewer.WindowId)
```

## 关键架构决策

| 决策 | 理由 |
|------|------|
| AssetEditorManager 放在 Assets 层 | Asset↔Window 映射属于资产打开逻辑 |
| TextureAssetOpener 构造函数注入 | 复用 AssetOpenerRegistry 的 DI 机制 |
| 后台 Task.Run 加载资产 | LoadAssetSync 读磁盘，不应阻塞 UI 线程 |
| Assets → ImGuiEx 引用 | Assets 是 Level 3，ImGuiEx 是 Level 2，无循环依赖 |
| Assets → Runtime.Engine 引用 | TextureInterop 在此，也是低层模块 |
| ImTextureViewerWindow 不引用 GUID 类型 | 保持 ImGuiEx 层轻量，用 string 传递标识 |
| AssetEditorManager 在 Discover 之前注册 | 确保 TextureAssetOpener 构造函数 DI 能解析 |
| LoadSync 失败时自动 Import 再重试 | 资产可能未被导入（无 .nnasset），Opener 应自愈 |
