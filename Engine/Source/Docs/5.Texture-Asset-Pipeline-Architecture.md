## 一、问题陈述

### 1.1 当前现状

当前 `NNRuntimeAssetLegacy` 中的 `TextureAsset` 存在严重耦合问题：

```cpp
// Engine/Source/Runtime/NNRuntimeAssetLegacy/Include/TextureAsset.h
struct TextureAsset : VGAsset
{
    uint32_t Width;
    uint32_t Height;
    ImageFormat Format;           // SDL_PixelFormat 1:1 映射
    std::vector<uint8_t> Data;    // CPU pixels
    void* UserPtr;                // SDL_Surface* — 强耦合
    FreeCallback FreeFunc;        // SDL_DestroySurface
};
```

`TextureAssetLoader::Read()` 内部：

```cpp
// Engine/Source/Runtime/NNRuntimeAssetLegacy/Source/TextureAsset.cpp
#include <SDL3_image/SDL_image.h>
SDL_Surface* surface = IMG_LoadTyped_IO(io, true, ext);
```

### 1.2 核心问题

| 问题 | 影响 |
|------|------|
| `void* UserPtr` 持有 `SDL_Surface*` | Asset 层与 SDL 强耦合 |
| `ImageFormat` 枚举直接映射 `SDL_PixelFormat` | 跨 API 不可移植 |
| GPU lifetime 混入 Asset 结构 | 无法支持 Diligent / Vulkan / Metal |
| 无 GPU Resource 抽象 | 无法 Streaming / Residency / Hot Reload |
| 无 ImGui Handle 抽象 | Editor 直接依赖 OpenGL `GLuint` |
| `NNEngineLegacy::Texture2D` 包含 `Ref<VGFX::ITexture>` | Legacy 与 RHI 耦合 |

### 1.3 目标

将旧的单一 `TextureAsset` 拆分为工业级分层：

```
源文件（PNG/JPG/HDR/TGA/BMP）
    ↓
Editor Importer (stb_image decode)
    ↓
TextureSourceAsset (CPU)
    ↓
RenderAssetManager (CPU → GPU upload)
    ↓
TextureResource (GPU)
    ↓
RHI Texture (OpenGL GLuint / Diligent ITexture* / VkImage)
    ↓
ImGui ImTextureID / SpriteRenderer / Material
```

## 二、工业级分层架构目标

### 2.1 四层分离

```
┌─────────────────────────────────────────────────┐
│ Layer 4: 使用层                                  │
│   ImGui.Image() / SpriteRenderer / Material      │
├─────────────────────────────────────────────────┤
│ Layer 3: GPU Resource 层                         │
│   NNTextureResource                              │
│   包含 RHITextureHandle + SRV + Desc + State     │
├─────────────────────────────────────────────────┤
│ Layer 2: RHI 抽象层                              │
│   NNRuntimeRHI (VGFX::ITexture)                  │
│   OpenGL backend / Diligent backend / Vulkan     │
├─────────────────────────────────────────────────┤
│ Layer 1: CPU Source Asset 层                     │
│   NNTextureSourceAsset                           │
│   纯 CPU pixels，无 GPU API                      │
│   stb_image decode 产物                          │
├─────────────────────────────────────────────────┤
│ Layer 0: 编码层                                  │
│   NNRuntimeImageCodec                            │
│   stb_image wrapper                              │
│   PNG/JPG/TGA/BMP/HDR → CPU RGBA8               │
└─────────────────────────────────────────────────┘
```

### 2.2 核心原则

1. **CPU Asset 不允许 GPU API** — 无 OpenGL、无 Diligent、无 `ImTextureID`
2. **GPU Resource 不暴露原生 Handle** — 外部只通过 `uint64` Handle 访问
3. **ImGui 不直接依赖 OpenGL** — 通过 `GetImGuiTextureHandle()` 抽象
4. **SpriteRenderer 只持有 Asset Handle** — 不持有 GPU Handle
5. **RHI 后端可替换** — OpenGL / Diligent / Vulkan / Metal 热切换

## 三、模块清单与职责

### 3.1 新增模块

| # | 模块名 | CMake Target | 类型 | 职责 |
|---|--------|-------------|------|------|
| 1 | `NNRuntimeImageCodec` | `NevernessRuntime-ImageCodec` | STATIC | stb_image 解码，PNG/JPG/TGA/BMP/HDR → CPU RGBA8 |
| 2 | `NNRuntimeRenderAssets` | `NevernessRuntime-RenderAssets` | STATIC | GPU Texture Resource 管理，CPU→GPU 生命周期 |

### 3.2 模块职责详述

#### 3.2.1 NNRuntimeImageCodec

纯 Codec 层。不允许 GPU API。

```
职责：
  - stb_image decode
  - stb_image_write encode
  - 支持 PNG/JPG/TGA/BMP/HDR
  - 返回 NNDecodedImage（CPU RGBA8 buffer）
  - Mipmap 生成（CPU 侧，stb_image_resize）

禁止：
  - OpenGL API
  - Diligent API
  - SDL_Surface
  - SDL_image
  - ImTextureID
  - GPU 资源管理
```

#### 3.2.2 NNRuntimeRenderAssets

CPU Asset → GPU Resource 管理层。

```
职责：
  - NNTextureSourceAsset（CPU 资产抽象）
  - NNTextureResource（GPU 资源抽象）
  - NNRenderAssetManager（CPU→GPU 生命周期管理）
  - ImGui Handle 抽象
  - Texture 缓存（AssetHandle → TextureResource 映射）
  - Texture 加载 / 卸载 / 重载
  - GPU Upload 队列（异步）
  - Residency 管理（预留）

依赖：
  - NNRuntimeAsset（Handle 系统）
  - NNRuntimeRHI（VGFX::ITexture 创建）
  - NNRuntimeImageCodec（解码）

禁止：
  - 直接暴露 GLuint / ITexture*
  - SDL_Surface
  - stb_image（委托给 ImageCodec）
```

### 3.3 现有模块角色

| 现有模块 | 角色 |
|----------|------|
| `NNRuntimeAsset` | Handle 系统、AssetManager、.nnasset 格式、缓存 |
| `NNAssetRegistry` | GUID↔VirtualPath、依赖图 |
| `NNAssetCooker` | 打包 .nnpack |
| `NNRuntimeRHI` | GPU API 抽象（VGFX::ITexture → OpenGL Texture2D） |
| `NNRuntimeManaged` | C# ↔ C++ ABI 桥接 |
| `Neverness.Editor.Assets` | Editor 导入管线、TextureImporter |

## 六、核心类定义

### 6.1 NNDecodedImage（CPU 解码结果）

```cpp
// Engine/Source/Runtime/NNRuntimeImageCodec/Include/NNDecodedImage.h

#pragma once
#include <cstdint>
#include <vector>

namespace NN::Runtime::ImageCodec
{

/// 纹素格式（引擎级，不依赖任何 API）
enum class NNImageFormat : uint32_t
{
    Unknown = 0,
    R8,
    RG8,
    RGB8,
    RGBA8,
    R16F,
    RG16F,
    RGB16F,
    RGBA16F,
    R32F,
    RG32F,
    RGB32F,
    RGBA32F,
};

/// 解码后的 CPU 图像数据
/// 纯 CPU，不允许任何 GPU API
struct NNDecodedImage
{
    uint32_t Width = 0;
    uint32_t Height = 0;
    uint32_t Channels = 0;
    NNImageFormat Format = NNImageFormat::Unknown;
    std::vector<uint8_t> Pixels;   // 行优先，紧密排列，无 padding

    bool IsValid() const { return !Pixels.empty() && Width > 0 && Height > 0; }
    size_t GetByteSize() const { return Pixels.size(); }

    void Reset()
    {
        Width = Height = Channels = 0;
        Format = NNImageFormat::Unknown;
        Pixels.clear();
    }
};

} // namespace NN::Runtime::ImageCodec
```

### 6.2 NNImageCodec（解码器）

```cpp
// Engine/Source/Runtime/NNRuntimeImageCodec/Include/NNImageCodec.h

#pragma once
#include "NNDecodedImage.h"
#include <span>
#include <string_view>

namespace NN::Runtime::ImageCodec
{

/// 纯函数式解码接口
/// 无状态，线程安全（stb_image 本身线程安全）
namespace ImageCodec
{
    /// 从内存解码图像
    /// @param data 压缩图像数据（PNG/JPG/TGA/BMP/HDR）
    /// @param desiredChannels 期望通道数（0=自动，3=RGB，4=RGBA）

    NNDecodedImage DecodeFromMemory(

        uint32_t desiredChannels = 0
    );

    /// 从文件路径解码
    NNDecodedImage DecodeFromFile(
        std::string_view filePath,
        uint32_t desiredChannels = 0
    );

    /// 生成 mipmap（CPU 侧）
    /// @param source 源图像（必须是 RGBA8）

    std::vector<uint8_t> GenerateMipsCPU(
        const NNDecodedImage& source,
        uint32_t mipLevels
    );

    /// 判断格式是否为 HDR（float 数据）
    bool IsHDRFormat(NNImageFormat format);

    /// 计算给定宽高的最大 mip 层数
    uint32_t CalculateMipCount(uint32_t width, uint32_t height);
}

} // namespace NN::Runtime::ImageCodec
```

```cpp
    RG32_Float,
    RGB32_Float,

    BC3_UNorm,        // DXT5
    BC7_UNorm,        // 高质量
    ETC2_RGB8,        // 移动端
    ETC2_RGBA8,
    ASTC_4x4,         // 移动端高质量

    Count
};

/// 从 NNImageFormat 映射到 NNTextureFormat
NNTextureFormat ToTextureFormat(uint32_t imageFormat);

/// 获取每个纹素的字节数（未压缩格式）

} // namespace NN::Runtime::Render
```

### 6.5 NNTextureResource（GPU 资源）

```cpp
// Engine/Source/Runtime/NNRuntimeRenderAssets/Include/NNTextureResource.h

#pragma once
#include "NNTextureFormat.h"
#include <cstdint>
#include <memory>

namespace NN::Runtime::Render
{

/// GPU Texture 描述
struct NNTextureDesc
{
    uint32_t Width = 0;
    uint32_t Height = 0;
    uint32_t MipCount = 1;
    NNTextureFormat Format = NNTextureFormat::RGBA8_UNorm;
    bool IsSRGB = false;
    bool GenerateMips = false;     // GPU 侧生成 mipmap
    bool IsRenderTarget = false;   // 可作为 RT 使用
};

/// Residency 状态（Streaming 预留）
enum class NNTextureResidency : uint8_t
{
    NotLoaded = 0,    // 未加载
    Loading,          // 加载中
    Resident,         // 常驻 GPU
    Evicted,          // 已驱逐
    Error,            // 加载失败
};

/// GPU 纹理资源
/// 封装 RHI Texture，不暴露原生 Handle
class NNTextureResource
{
public:
    NNTextureResource() = default;
    ~NNTextureResource();

    // 禁止拷贝
    NNTextureResource(const NNTextureResource&) = delete;
    NNTextureResource& operator=(const NNTextureResource&) = delete;

    // 允许移动
    NNTextureResource(NNTextureResource&& other) noexcept;
    NNTextureResource& operator=(NNTextureResource&& other) noexcept;

    const NNTextureDesc& GetDesc() const { return m_Desc; }
    NNTextureResidency GetResidency() const { return m_Residency; }
    uint64_t GetLastUsedFrame() const { return m_LastUsedFrame; }

    /// 获取内部 RHI Texture 指针（仅内部使用）
    /// 不对外暴露
    void* GetRHITextureInternal() const { return m_RHITexture; }

    /// 获取 ImGui 兼容的 Texture Handle
    /// OpenGL: GLuint cast to uint64
    /// Diligent: ITextureView* cast to uint64
    /// Metal: id<MTLTexture> cast to uint64
    uint64_t GetImGuiHandle() const;

    bool IsResident() const { return m_Residency == NNTextureResidency::Resident; }

private:
    friend class NNRenderAssetManager;

    NNTextureDesc m_Desc;
    void* m_RHITexture = nullptr;       // VGFX::ITexture*，不暴露类型
    void* m_RHIShaderResourceView = nullptr;  // SRV（Diligent 用）
    uint64_t m_LastUsedFrame = 0;
    NNTextureResidency m_Residency = NNTextureResidency::NotLoaded;
};

} // namespace NN::Runtime::Render
```

### 6.6 NNRenderAssetManager（生命周期管理）

```cpp
// Engine/Source/Runtime/NNRuntimeRenderAssets/Include/NNRenderAssetManager.h

#pragma once
```

### 6.7 NNTextureCache（GPU 资源缓存）

## 九、Runtime Load 流程

### 9.1 从 .nnasset 加载 Texture2D 的完整流程

```
步骤 1: AssetManager.LoadSync(handle)
  └─ 查找 AssetEntry (GUID, typeId)
  └─ 确认 typeId == NN_TYPE_ID_TEXTURE_2D
  └─ 读取 .nnasset 文件到内存 buffer

步骤 2: 解析 NNAssetHeader (96 bytes)
  └─ 验证 magic == 'NNAS'
  └─ 验证 typeId == 1 (Texture2D)
  └─ 读取 blob descriptors

  └─ Blob 0: MipLevel0 (RGBA8 pixels)
  └─ Blob 1: MipLevel1 (optional)
  └─ ...
  └─ TypeInfo: width, height, format, mipCount, srgb, hasAlpha

步骤 4: 构造 NNTextureSourceAsset
  └─ source.SetFromDecodedImage(width, height, format, pixels, srgb, hasAlpha)

  └─ 构造 VGFX::TextureDesc
  └─ VGFX::CreateTextureFromMemory(desc) → ITexture*
  └─ 包装为 NNTextureResource
  └─ 缓存

步骤 6: 返回 AssetHandle
  └─ 外部通过 GetTextureResource(handle) 使用
  └─ 通过 GetImGuiTextureHandle(handle) 获取 ImTextureID
```

## 十、ImGui Handle 抽象

### 10.1 核心问题

ImGui 的 `ImTextureID` 在不同后端含义不同：

| 后端 | ImTextureID 实际类型 |
|------|---------------------|
| OpenGL | `GLuint`（texture id） |
| Diligent | `ITextureView*`（SRV pointer） |
| Vulkan | `VkDescriptorSet` |
| Metal | `id<MTLTexture>` |

### 10.2 解决方案

`NNTextureResource::GetImGuiHandle()` 返回 `uint64`，内部根据当前 RHI 后端决定：

```cpp
// Engine/Source/Runtime/NNRuntimeRenderAssets/Private/NNTextureResource.cpp

uint64_t NNTextureResource::GetImGuiHandle() const
{
    if (!m_RHITexture)
        return 0;

#if NN_RHI_OPENGL
    // OpenGL: ITexture → GLuint
    auto* glTex = static_cast<VGFX::OpenGL::Texture2D*>(m_RHITexture);
    return static_cast<uint64_t>(glTex->GetNativeHandle());

#elif NN_RHI_DILIGENT
    // Diligent: SRV pointer → uint64
    auto* srv = static_cast<ITextureView*>(m_RHIShaderResourceView);
    return reinterpret_cast<uint64_t>(srv);

#elif NN_RHI_VULKAN
    // Vulkan: VkDescriptorSet → uint64
    auto descSet = reinterpret_cast<VkDescriptorSet>(m_RHIShaderResourceView);
    return static_cast<uint64_t>(descSet);

#elif NN_RHI_METAL
    // Metal: id<MTLTexture> → uint64
    return reinterpret_cast<uint64_t>(m_RHITexture);

#else
    return 0;
#endif
}
```

### 10.3 C++ ImGui 使用

```cpp
// Editor 或 Debug 界面中
uint64_t texHandle = NNRenderAssetManager::Get()
    .GetImGuiTextureHandle(myTextureAssetHandle);

ImGui::Image(
    reinterpret_cast<ImTextureID>(texHandle),
    ImVec2(width, height)
);
```

### 10.4 关键设计原则

**API 必须命名为 `GetImGuiTextureHandle()`**，不能命名为：
- `GetOpenGLTexture()` — 后端锁定
- `GetGLTextureID()` — 后端锁定
- `GetNativeTextureID()` — 含义模糊

命名 `GetImGuiTextureHandle()` 明确了：
- 返回值专门用于 ImGui
- 不承诺底层 API 类型
- 未来后端切换无需改调用方

```cpp
// Engine/Source/Runtime/NNRuntimeRenderAssets/Private/NNRenderAssetApi.cpp

#include "NNRenderAssetApi.h"
#include "NNRenderAssetManager.h"
#include "NNTextureSourceAsset.h"
#include <cstring>

using namespace NN::Runtime::Render;

static uint64_t Api_GetImGuiTextureHandle(uint64_t handle)
{
    AssetHandleT<void> ah;
    std::memcpy(&ah, &handle, sizeof(uint64_t));
    return NNRenderAssetManager::Get().GetImGuiTextureHandle(ah);
}

static uint64_t Api_CreateTextureFromPixels(
    uint32_t w, uint32_t h,
    const uint8_t* pixels, size_t size,
    int isSRGB)
{
    NNTextureSourceAsset source;
    std::vector<uint8_t> pixelCopy(pixels, pixels + size);
    source.SetFromDecodedImage(w, h, NNTextureFormat::RGBA8_UNorm,
                                std::move(pixelCopy),
                                isSRGB != 0, true);

    auto* resource = NNRenderAssetManager::Get().CreateTextureFromSource(source);
    if (!resource) return 0;

    // 返回一个内部 handle（需要与 AssetManager 集成）
    // 这里简化：实际应注册到 HandleTable
    return reinterpret_cast<uint64_t>(resource);  // Phase 1 临时方案
}

static void Api_ReleaseTexture(uint64_t handle)
{
    AssetHandleT<void> ah;
    std::memcpy(&ah, &handle, sizeof(uint64_t));
    NNRenderAssetManager::Get().ReleaseTexture(ah);
}

{
    AssetHandleT<void> ah;
    std::memcpy(&ah, &handle, sizeof(uint64_t));
    NNRenderAssetManager::Get().ReloadTexture(ah);
}

static int Api_GetTextureDesc(uint64_t handle, uint32_t* w, uint32_t* h)
{
    AssetHandleT<void> ah;
    std::memcpy(&ah, &handle, sizeof(uint64_t));
    auto* res = NNRenderAssetManager::Get().GetTextureResource(ah);
    if (!res) return 0;
    if (w) *w = res->GetDesc().Width;
    if (h) *h = res->GetDesc().Height;
    return 1;
}

{

    std::memcpy(&ah, &handle, sizeof(uint64_t));
    auto* res = NNRenderAssetManager::Get().GetTextureResource(ah);
    return res && res->IsResident() ? 1 : 0;
}
```

```csharp
// Engine/Source/Managed/Runtime/Neverness.Runtime.Render/TextureInterop.cs

using System;
using System.Runtime.InteropServices;

namespace Neverness.Runtime.Render
{
    /// <summary>
    /// Render Asset API 函数表的 C# 映射
    /// </summary>
    [StructLayout(LayoutKind.Sequential)]
    public unsafe struct RenderAssetApiTable
    {
        public uint ApiVersion;

        public delegate* unmanaged<ulong, ulong> GetImGuiTextureHandle;
        public delegate* unmanaged<uint, uint, byte*, nuint, int, ulong> CreateTextureFromPixels;
        public delegate* unmanaged<ulong, void> ReleaseTexture;
        public delegate* unmanaged<ulong, void> ReloadTexture;
        public delegate* unmanaged<ulong, uint*, uint*, int> GetTextureDesc;
        public delegate* unmanaged<ulong, int> IsTextureResident;
    }

    /// <summary>
    /// Texture 互操作静态类
    /// </summary>
    public static class TextureInterop
    {
        private static RenderAssetApiTable* s_Api;

        /// <summary>
        /// 初始化（由 Engine 初始化时调用）
        /// </summary>
        public static unsafe void Initialize(RenderAssetApiTable* api)
        {
            s_Api = api;
        }

        /// <summary>
        /// 获取 Texture 的 ImGui Handle
        /// </summary>
        /// <param name="textureAsset">AssetManager 返回的 handle</param>
        /// <returns>ImGui 兼容的 texture handle</returns>
        public static unsafe ulong GetImGuiTextureHandle(ulong textureAsset)
        {
            if (s_Api == null || textureAsset == 0)
                return 0;
            return s_Api->GetImGuiTextureHandle(textureAsset);
        }

        /// <summary>
        /// 从像素数据创建 Texture
        /// </summary>
        public static unsafe ulong CreateTexture(
            uint width, uint height,
            ReadOnlySpan<byte> pixels,
            bool isSRGB = false)
        {
            if (s_Api == null)
                return 0;

            fixed (byte* pPixels = pixels)
            {
                return s_Api->CreateTextureFromPixels(
                    width, height,
                    pPixels, (nuint)pixels.Length,
                    isSRGB ? 1 : 0
                );
            }
        }

        /// <summary>
        /// 释放 Texture GPU 资源
        /// </summary>
        public static unsafe void ReleaseTexture(ulong textureAsset)
        {
            if (s_Api != null && textureAsset != 0)
                s_Api->ReleaseTexture(textureAsset);
        }

        /// <summary>
        /// 重载 Texture（Hot Reload）
        /// </summary>
        public static unsafe void ReloadTexture(ulong textureAsset)
        {
            if (s_Api != null && textureAsset != 0)
                s_Api->ReloadTexture(textureAsset);
        }

        /// <summary>
        /// 获取 Texture 尺寸
        /// </summary>
        public static unsafe (uint Width, uint Height) GetTextureSize(ulong textureAsset)
        {
            if (s_Api == null || textureAsset == 0)
                return (0, 0);

            uint w = 0, h = 0;
            if (s_Api->GetTextureDesc(textureAsset, &w, &h) != 0)
                return (w, h);
            return (0, 0);
        }

        /// <summary>
        /// Texture 是否已加载到 GPU
        /// </summary>
        public static unsafe bool IsTextureResident(ulong textureAsset)
        {
            if (s_Api == null || textureAsset == 0)
                return false;
            return s_Api->IsTextureResident(textureAsset) != 0;
        }
    }
}
```

### 14.2 为什么 TextureAsset 正确

`TextureAsset = AssetHandle` 而不是 `GPU Handle`，原因：

| 场景 | Asset Handle | GPU Handle |
|------|-------------|------------|
| GPU 资源被驱逐 | handle 有效，重新加载 | handle 失效 |
| Hot Reload | handle 不变 | handle 重建 |
| Streaming | handle 不变 | mip 层级变化 |
| 跨 API 切换 | handle 不变 | 完全不同 |
| Save/Load Scene | handle 序列化 | 不能序列化 |
| Prefab 引用 | handle 稳定 | 不稳定 |

## 十七、Hot Reload 方案

### 17.1 完整流程

```
FileWatcher 检测 hero.png 变化
    │
    ▼
Editor ImportPipeline
    ├─ 重新读取 hero.png
    ├─ stb_image decode
    ├─ 序列化到 .nnasset
    └─ 通知 Runtime
        │
        ▼
    NNAssetManager::ReloadAsset(handle)
        │
        ▼
    NNRenderAssetManager::ReloadTexture(handle)
        ├─ Release old VGFX::ITexture
        ├─ 从新 .nnasset 解析 NNTextureSourceAsset
        ├─ CreateTextureFromSource(source)
        ├─ 更新 m_Cache[handle]
        └─ 所有引用自动获得新 GPU texture
            │
            ▼
        SpriteRenderer.TextureAsset handle 不变
        下一帧渲染使用新 texture
```

### 17.2 关键点

- **Asset Handle 不变** — 所有持有 `TextureAsset = handle` 的组件无需更新
- **GPU Resource 替换** — 缓存中重建 `NNTextureResource`
- **ImGui Handle 更新** — `GetImGuiHandle()` 返回新的 `ImTextureID`
- **线程安全** — `m_Cache` 操作受 `m_Mutex` 保护

## 十八、工业级注意事项

### 18.1 禁止事项

| 禁止 | 原因 |
|------|------|
| 禁止在 `NNTextureSourceAsset` 中使用 `GLuint` | CPU Asset 必须与 GPU API 无关 |
| 禁止在 `NNRuntimeImageCodec` 中依赖 `SDL_image` | Codec 层必须纯 CPU，stb_image 足够 |
| 禁止暴露 `VGFX::ITexture*` 到 Asset 层 | 封装必须通过 `NNTextureResource` |
| 禁止命名 `GetOpenGLTexture()` | 后端锁定，未来不可扩展 |
| 禁止在 `SpriteRenderer` 中存储 `GLuint` | 跨 API 不可移植，资源重建后失效 |
| 禁止在 `NNRenderAssetManager` 中直接调用 `glTexImage2D` | 必须通过 `VGFX::CreateTextureFromMemory()` |
| 禁止 `NNTextureResource` 拷贝 | GPU 资源唯一所有权 |

### 18.2 必须事项

| 必须 | 原因 |
|------|------|
| 必须使用 `stb_image` 解码 | 跨平台，无 SDL 依赖 |
| 必须通过 `VGFX::CreateTextureFromMemory()` 创建 | RHI 抽象 |
| 必须使用 `uint64` Handle 传递 Texture Identity | ABI 稳定性 |
| 必须在 `NNTextureResource` 中缓存 `LastUsedFrame` | LRU 驱逐 / Streaming |
| 必须使用 `mutex` 保护 `m_Cache` | 多线程安全 |
| 必须预留 `NNTextureResidency` 状态 | 未来 Streaming |
| 必须支持序列化 / 反序列化 | .nnasset 格式 |

### 18.3 性能考虑

| 考量 | 方案 |
|------|------|
| GPU 上传应在渲染线程 | 上传队列 + Flush |
| 大纹理异步加载 | Streaming Manager |
| LRU 驱逐避免 OOM | `NNTextureCache::EvictToTarget()` |
| Handle 查询 O(1) | `unordered_map<uint64_t, CacheEntry>` |
| 锁粒度 | Cache 级别 mutex，非全局 |
| Mipmap 可选 GPU 生成 | `glGenerateMipmap` / 自动 mip chain |

### 18.4 可扩展性

| 扩展点 | 设计 |
|--------|------|
| 新 RHI 后端 | 实现 `VGFX::CreateTextureFromMemory()` 的新后端 |
| 压缩纹理 | `NNTextureFormat` 增加 BC/ETC2/ASTC，ImageCodec 支持 DDS/KTX |
| Texture Array | `NNTextureDesc` 增加 `ArraySize` |
| Cube Map | `NNTextureDesc` 增加 `IsCubeMap` |
| Virtual Texture | `NNRenderAssetManager` 增加 VT 页表 |
| Compute Shader | `NNTextureResource` 增加 UAV Handle |

## 二十、最终架构总结

### 20.1 完整模块层次

```
┌──────────────────────────────────────────────────────────────┐
│                     Editor (C#)                              │
│                                                              │
│  TextureImporter → ImportPipeline → EditorAssetDatabase      │
│  TextureInspector → ImGui.Image()                            │
│  SpriteRendererInspector                                     │
└─────────────────────────┬────────────────────────────────────┘
                          │ PInvoke
┌─────────────────────────▼────────────────────────────────────┐
│                   NNRuntimeManaged                           │
│                                                              │
│  TextureInterop.GetImGuiTextureHandle()                      │
│  TextureInterop.CreateTexture()                              │
│  RenderAssetApiTable (delegate* unmanaged)                   │
└─────────────────────────┬────────────────────────────────────┘
                          │ C ABI
┌─────────────────────────▼────────────────────────────────────┐
│              NNRuntimeRenderAssets (STATIC)                  │
│                                                              │
│  NNTextureSourceAsset    — CPU pixels, 无 GPU API            │
│  NNTextureResource       — GPU texture, 有 Residency 状态   │
│  NNRenderAssetManager    — CPU → GPU 生命周期管理            │
│  NNTextureCache          — LRU 缓存                         │
│  NNRenderAssetApi        — C ABI 函数表                      │
│  NNTextureFormat         — 引擎级格式枚举                    │
└──┬──────────────────┬────────────────────┬───────────────────┘
   │                  │                    │
   ▼                  ▼                    ▼
┌──────────┐  ┌───────────────┐  ┌──────────────────┐
│NNRuntime │  │ NNRuntimeRHI  │  │ NNRuntimeAsset   │
│ImageCodec│  │ (VGFX::ITexture│  │ (Handle/Manager) │
│(stb_image│  │  OpenGL impl) │  │                  │
│)         │  │               │  │                  │
└──────────┘  └───────────────┘  └──────────────────┘
```

### 20.2 关键 API 一览

| API | 层级 | 用途 |
|-----|------|------|
| `NNImageCodec::DecodeFromMemory()` | Codec | PNG/JPG → CPU RGBA8 |
| `NNTextureSourceAsset::Serialize()` | CPU Asset | 序列化到 .nnasset blob |
| `NNRenderAssetManager::GetTextureResource()` | GPU Manager | Handle → GPU Texture |
| `NNRenderAssetManager::GetImGuiTextureHandle()` | GPU Manager | Handle → ImTextureID |
| `NNTextureResource::GetImGuiHandle()` | GPU Resource | 后端特定的 ImTextureID |
| `VGFX::CreateTextureFromMemory()` | RHI | 创建 RHI Texture |
| `NNRenderAssetApi.getImGuiTextureHandle()` | C ABI | C# 可调用 |
| `TextureInterop.GetImGuiTextureHandle()` | C# | Editor ImGui 使用 |

### 20.3 与现有架构的对齐

```
现有 Asset Pipeline (Doc 1)
├── .nnasset 格式 ← TextureSourceAsset::Serialize() 产出
├── Import Pipeline ← TextureImporter 使用 ImageCodec::DecodeFromMemory()
├── Streaming Manager ← RenderAssetManager 扩展
├── Asset Cache ← TextureCache (GPU 侧独立缓存)
├── C ABI ← NNRenderAssetApi 子表
└── Hot Reload ← RenderAssetManager::ReloadTexture()
```

新模块完全对齐现有架构设计，不冲突，不重复。

### 20.4 最终目标

```
源文件（PNG/JPG/HDR）
    ↓ stb_image（NNRuntimeImageCodec）
CPU Texture Source Asset（NNTextureSourceAsset）
    ↓ Serialize（.nnasset format）
Disk（Library/Imported/）
    ↓ Deserialize（NNAssetManager）
CPU Texture Source Asset（运行时）
    ↓ GPU Upload（VGFX::CreateTextureFromMemory）
GPU Texture Resource（NNTextureResource）
    ↓ GetImGuiHandle()
ImTextureID → ImGui.Image() / SpriteRenderer / Material
```

**这就是 Unity / Unreal 工业级 Texture Asset Pipeline。**
