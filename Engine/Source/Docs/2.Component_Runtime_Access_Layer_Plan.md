# Component Runtime Access Layer — 消灭所有 typeid/if 链

## Context

当前 Neverness Engine 的 ECS 系统存在**三处完全相同的 if/typeid 硬编码分发链**：

1. **SceneSubsystem.cpp `GetComponentRead`/`GetComponentWrite`** (行 129-184) — 3 个 `std::type_index(typeid(...))` 链
2. **SceneSubsystem.cpp `QueryEntities`** (行 436-499) — 3 个 if 链
3. **NNSceneSerializer.cpp `Deserialize`** (行 418-461) — 3 个 if 链 + 1 个空的 generic fallback

每增加一个组件类型，需要修改 **5+ 个位置**。这在 C# 扩展组件、热加载、插件架构下完全不可持续。

**目标**：将 `NNComponentTypeDesc` 从"字段描述结构"升级为"运行时类型对象（Runtime Type Object）"，每个描述符自带 type-erased 的 ECS 操作函数指针，彻底消灭所有 if/typeid 链。

---

## Phase 1: 扩展 NNComponentTypeDesc — 添加 Runtime Function Pointers

**文件**: `Engine/Source/Runtime/NNRuntimeScene/Include/Reflection/NNComponentRegistry.h`

在 `NNComponentTypeDesc` 结构体中新增 6 个函数指针（C-compatible，无 `std::function` 开销）：

```cpp
struct NN_RUNTIME_SCENE_API NNComponentTypeDesc
{
    // ... 现有字段不变 (TypeId, NameHash, TypeIndex, NameUtf8, SizeBytes, Fields, SerializeFn) ...

    // ── Runtime Type-Erased ECS Access（Phase 5 新增）──

    /// 获取组件可写指针（无组件时返回 nullptr）
    void* (*GetComponentPtrFn)(NNRuntimeScene* scene, NNEntity entity) = nullptr;

    /// 获取组件只读指针（无组件时返回 nullptr）
    const void* (*GetComponentConstPtrFn)(const NNRuntimeScene* scene, NNEntity entity) = nullptr;

    /// 查询实体是否拥有此组件
    bool (*HasComponentFn)(const NNRuntimeScene* scene, NNEntity entity) = nullptr;

    /// 动态添加组件（成功返回 true）
    bool (*AddComponentFn)(NNRuntimeScene* scene, NNEntity entity) = nullptr;

    /// 动态移除组件（成功返回 true）
    bool (*RemoveComponentFn)(NNRuntimeScene* scene, NNEntity entity) = nullptr;

    /// 遍历拥有此组件的所有实体（回调式，用于 QueryEntities）
    void (*ForEachEntityFn)(NNRuntimeScene* scene,
        void(*callback)(NNEntity entity, void* component, void* userData),
        void* userData) = nullptr;

    /// 反序列化后处理回调（可选，如 Relationship 需要调用 SetParent）
    void (*PostDeserializeFn)(NNRuntimeScene& scene, NNEntity entity) = nullptr;
};
```

**设计决策**：
- 使用 C 风格函数指针而非 `std::function`，确保 DLL 边界安全（热加载/插件场景）
- 所有函数指针默认 nullptr，向后兼容（旧代码仍可注册不带函数指针的描述符）
- `ForEachEntityFn` 使用回调式迭代器（非返回 view 对象），避免暴露 entt 内部类型

## Phase 2: 自动绑定函数指针 — 扩展 NN_REGISTER_COMPONENT 宏

**文件**: `Engine/Source/Runtime/NNRuntimeScene/Include/Reflection/NNComponentRegistry.h`

扩展 `NN_REGISTER_COMPONENT` 宏，在注册时自动绑定 entt 操作到函数指针。

宏内每个 lambda 捕获编译期类型 `Type`，运行时通过函数指针调用——零虚表开销。

关键绑定：
```
GetComponentPtrFn    → scene->TryGet<Type>(entity)
GetComponentConstPtrFn → scene->TryGet<Type>(entity) (const)
HasComponentFn       → scene->Has<Type>(entity)
AddComponentFn       → scene->Has ? false : scene->Emplace<Type>, true
RemoveComponentFn    → scene->Remove<Type>(entity)
ForEachEntityFn      → registry.view<Type>() 遍历
```

所有 lambda 在 static 初始化期构造，仅依赖函数参数（无外部状态捕获）。

同时新增 `NN_REGISTER_COMPONENT_WITH_POST_DESERIALIZE` 宏变体，支持为 Relationship 等特殊组件注册 `PostDeserializeFn`。

## Phase 5: 消灭 NNSceneSerializer.cpp 中的 if/typeid 链

**文件**: `Engine/Source/Runtime/NNRuntimeScene/Source/Serialization/NNSceneSerializer.cpp`

### 5.1: Serialize — 统一为完全 registry-driven

消除 "Phase A (hardcoded) / Phase B (ForEachDescriptor + skip built-in)" 的分治逻辑：

```cpp
registry.ForEachDescriptor([&](const NNComponentTypeDesc& desc)
{
    if (!desc.GetComponentConstPtrFn || desc.Fields.empty()) return;
    const void* ptr = desc.GetComponentConstPtrFn(&scene, handle);
    if (ptr == nullptr) return;
    // SerializeComponentBlob + append
});
```

彻底移除所有 `typeid` 比较和 "skip built-in" 逻辑。

### 5.2: Deserialize — 实现 generic fallback

**Before**: 3 个 `if (desc->TypeIndex == typeid(...))` + 1 个空的 generic fallback 注释
**After**: 完全通用的反序列化路径

```cpp
if (desc->GetComponentPtrFn && !desc->Fields.empty())
{
    // 1. 分配临时缓冲区
    std::vector<uint8_t> temp(desc->SizeBytes);
    // 2. 反序列化 blob 到临时缓冲区（已有 DeserializeComponentBlob）
    std::size_t blobOffset = 0;
    if (DeserializeComponentBlob(*desc, temp.data(), pc.Blob, blobOffset, archiveToHandle))
    {
        // 3. 从 desc 获取 ECS 写入指针并 memcpy
        void* dst = desc->GetComponentPtrFn(&scene, handle);
        if (dst) std::memcpy(dst, temp.data(), desc->SizeBytes);
        // 4. 调用可选的后处理回调（如 Relationship 的 SetParent）
        if (desc->PostDeserializeFn) desc->PostDeserializeFn(scene, handle);
    }
}
```

Transform 和 Tag 自动走通用路径（零特殊代码）。
Relationship 通过 `PostDeserializeFn` 回调处理 `SetParent` 逻辑。

## 受益效果

### 注册新组件 = 零修改旧代码

以后注册新组件只需：
```cpp
NN_REGISTER_COMPONENT(
    NNCameraComponent,
    "Camera",
    NN_FIELD(NNCameraComponent, Projection, UInt32),
    NN_FIELD(NNCameraComponent, NearPlane, Float),
    ...);
```

自动获得：
- `GetComponent` / `SetComponent` — C# 端可用
- `HasComponent` / `AddComponent` / `RemoveComponent` — 动态增删
- `QueryEntities` — 批量查询
- 序列化 / 反序列化 — 无需写任何序列化代码
- Inspector 编辑器 — C# 端 DetailInspector 自动绘制

### 未来扩展

- **C# 扩展组件**：C# 注册 `RegisterComponent(typeof(MyComp))` → Native registry 自动生成 runtime accessor
- **DLL 插件组件**：插件的 `NN_REGISTER_COMPONENT` 在 DLL 加载时自动注册到 global registry
- **热加载**：函数指针是 C-style，不依赖 vtable，DLL 重载后指针自动更新
- **网络复制**：添加 `ReplicateFn` 到描述符即可
