# Save Scene 功能实现

## 概述

实现 Editor 中 "File/Save Scene" 菜单项、工具栏 "Save Scene" 按钮和 Ctrl+S 快捷键的保存场景功能。

完整的序列化管线已存在：`SceneWorld.Save(vfsPath)` → `SceneNativeBridge.SerializeScene(handle, vfsPath)` → Native `SerializeScene`。
本次实现补充了**路径跟踪**、**命令注册**和**UI 接线**。

## 架构

```
用户触发（菜单/工具栏/Ctrl+S）
  → EditorMenuRegistry.ExecuteCommand("file.save")
    → EditorCommand.Execute()
      → SceneManager.SaveActiveScene()
        → SceneWorld.Save(AssetPath)
          → SceneNativeBridge.SerializeScene(handle, vfsPath)
            → Native api.SerializeScene(handle, path)
              → VGSC 二进制写入 VFS
```

## 详细变更

### 1. SceneWorld 添加 AssetPath

`SceneWorld` 新增 `AssetPath` 属性，跟踪场景的 VFS 资产路径：

- `LoadFromAsset` 工厂方法加载时自动设置 `AssetPath = vfsPath`
- `SaveSnapshot` 保存到热重载快照
- `RestoreFromSnapshot` 从快照恢复
- `HotReloadSnapshot` 新增 `AssetPath` 字段

### 2. SceneManager 添加保存方法

- `SaveActiveScene()`：保存激活场景到其 `AssetPath`，无路径时返回 `Invalid`
- `SaveActiveSceneAs(string vfsPath)`：保存到指定路径并更新 `AssetPath`

### 3. SceneEditorBridge 同步 EditorState

`OnSceneActivated` 回调中新增同步：
```csharp
var state = EditorCoreModule.Context.State;
state.CurrentScenePath = world.AssetPath;
state.CurrentSceneHandle = world.NativeHandle;
```

### 4. SceneModuleImp 注册 Save 命令

在 `Install` 方法中：
- 创建 `EditorCommand { Id = "file.save" }`，执行 `sceneManager.SaveActiveScene()`
- `CanExecute` 绑定 `sceneManager.HasActiveScene`
- 注册命令并覆盖 `File/Save Scene` 菜单项绑定命令

### 5. EditorMainWindow 接线

- 工具栏 "Save Scene" 按钮：`EditorMenuRegistry.ExecuteCommand("file.save")`
- `HandleKeyboardShortcuts()` 方法处理 Ctrl+S 全局快捷键
- 使用 `ImGui.IsKeyPressed` 防止重复触发

## 调用链路

```
用户点击 File/Save Scene（或 Ctrl+S 或工具栏按钮）
  → EditorMenuRegistry.ExecuteCommand("file.save")
    → EditorCommand.Execute()
      → SceneManager.SaveActiveScene()
        → SceneWorld.Save(AssetPath)
          → SceneNativeBridge.SerializeScene(handle, vfsPath)
            → Native SerializeScene
```
