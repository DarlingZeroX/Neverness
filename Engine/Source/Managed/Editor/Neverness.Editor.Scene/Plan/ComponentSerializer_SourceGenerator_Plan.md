# ComponentSerializer Source Generator 计划

## 问题

当前场景序列化/反序列化是手写 switch + 手动逐字段读写（`FrifloScene.Serialize/Deserialize`），每新增一个组件类型都要改两处 switch + 写一对 Write/Read 方法。`PrefabInstantiator.AddComponentFromData` 也只处理 3 个组件且忽略数据。

## 方案

用 Source Generator 自动生成组件序列化/反序列化代码。新增组件只需加一个 `[SerializableComponent]` 特性，Generator 自动扫描并生成。

---

## 1. 新增 `[Transient]` 特性

**文件**: `Engine/Source/Managed/Runtime/Neverness.Runtime.Engine/TransientAttribute.cs`

```csharp
/// <summary>
/// 标记字段为瞬态，序列化时跳过。
/// 用于运行时计算字段（如 WorldMatrix、RuntimePlayerId）。
/// </summary>
[AttributeUsage(AttributeTargets.Field, Inherited = false, AllowMultiple = false)]
public sealed class TransientAttribute : Attribute { }
```

**应用到现有组件**（标记不需要序列化的字段）：

| 组件 | 标记字段 |
|------|----------|
| TransformComponent | `WorldMatrix` |
| CameraComponent | `ProjectionMatrix`, `ViewMatrix` |
| ScriptComponent | `InstanceId`, `IsInitialized` |
| SpriteRendererComponent | `TextureRuntimeId` |
| AudioSourceComponent | `RuntimePlayerId` |
| VideoPlayerComponent | `RuntimePlayerId`, `VideoTextureId` |

TagComponent 和 RelationshipComponent 不参与 Generator 序列化（名字和层级在场景 JSON 顶层处理）。

## 2. 新增 `ComponentSerializerGenerator`

**文件**: `Engine/Source/Managed/Gameplay/Neverness.SourceGenerators/ComponentSerializerGenerator.cs`

复用现有 Generator 模式（`IIncrementalGenerator` + `CreateSyntaxProvider` + `Collect` + `StringBuilder`）。

### 扫描目标

- `struct` 类型
- 实现 `IComponent`（`Neverness.Runtime.Scene.IComponent`）
- 标记了 `[SerializableComponent]` 特性（或复用 `[ComponentId]`）

### 生成代码

输出文件: `ComponentSerializer_AutoGenerated.g.cs`

```csharp
// <auto-generated/>
namespace Neverness.Runtime.Scene.Internal;

public static class ComponentSerializer
{
    // ── 按类型名分发的序列化入口 ──
    public static void SerializeComponent<T>(ref T component, System.Text.Json.Utf8JsonWriter writer)
        where T : struct, IComponent
    {
        if (typeof(T) == typeof(TransformComponent))
            WriteTransformComponent(ref Unsafe.As<T, TransformComponent>(ref component), writer);
        else if (typeof(T) == typeof(CameraComponent))
            WriteCameraComponent(ref Unsafe.As<T, CameraComponent>(ref component), writer);
        // ... 其他组件
    }

    // ── 按类型名分发的反序列化入口 ──
    public static bool TryAddComponent(Entity entity, string typeName, System.Text.Json.JsonElement elem)
    {
        switch (typeName)
        {
            case "TransformComponent":
                entity.AddComponent(ReadTransformComponent(elem));
                return true;
            case "CameraComponent":
                entity.AddComponent(ReadCameraComponent(elem));
                return true;
            // ... 其他组件
            default:
                return false;
        }
    }

    // ── 每个组件一对 Write/Read 方法 ──

    private static void WriteTransformComponent(ref TransformComponent c, Utf8JsonWriter w)
    {
        w.WritePropertyName("TransformComponent");
        w.WriteStartObject();
        WriteVector3(w, "position", c.Position);
        WriteQuaternion(w, "rotation", c.Rotation);
        WriteVector3(w, "scale", c.Scale);
        // WorldMatrix 标记了 [Transient]，自动跳过
        w.WriteEndObject();
    }

    private static TransformComponent ReadTransformComponent(JsonElement elem)
    {
        var c = TransformComponent.Default;
        if (elem.TryGetProperty("position", out var p)) c.Position = ReadVector3(p);
        if (elem.TryGetProperty("rotation", out var r)) c.Rotation = ReadQuaternion(r);
        if (elem.TryGetProperty("scale", out var s)) c.Scale = ReadVector3(s);
        return c;
    }

    // ... 其他组件同理

    // ── 基础类型辅助方法（只生成一份）──
    private static void WriteVector3(Utf8JsonWriter w, string name, System.Numerics.Vector3 v) { ... }
    private static System.Numerics.Vector3 ReadVector3(JsonElement elem) { ... }
    private static void WriteQuaternion(Utf8JsonWriter w, string name, System.Numerics.Quaternion q) { ... }
    private static System.Numerics.Quaternion ReadQuaternion(JsonElement elem) { ... }
    private static void WriteNNGuid(Utf8JsonWriter w, string name, NNGuid guid) { ... }
    private static NNGuid ReadNNGuid(JsonElement elem) { ... }
}
```

### 字段扫描逻辑

对每个组件 struct：
1. 枚举所有 `public` 字段（`FieldInfo`，非 property）
2. 跳过标记了 `[Transient]` 的字段
3. 跳过 `fixed` 缓冲区字段（如 `TagComponent._name`）
4. 按字段类型生成对应的 Read/Write 调用：
   - `float` → `w.WriteNumber(name, v)` / `elem.GetSingle()`
   - `bool` → `w.WriteBoolean(name, v)` / `elem.GetBoolean()`
   - `uint` / `int` / `ulong` → `w.WriteNumber(name, v)` / `elem.GetUInt32()` 等
   - `Vector3` → `WriteVector3(w, name, v)` / `ReadVector3(elem)`
   - `Quaternion` → `WriteQuaternion(w, name, q)` / `ReadQuaternion(elem)`
   - `NNGuid` → `WriteNNGuid(w, name, g)` / `ReadNNGuid(elem)`
   - `enum` → `w.WriteString(name, v.ToString())` / `Enum.TryParse<T>(...)`
   - `Matrix4x4` → 跳过（计算字段，应标 `[Transient]`）

## 4. 集成到 FrifloScene

**文件**: `Private/Friflo/FrifloScene.cs`

替换手写序列化为调用生成代码：

```csharp
// Serialize 中：
foreach (var compProp in compsElem.EnumerateObject())
{
    ComponentSerializer.TryAddComponent(entity, compProp.Name, compProp.Value);
}

// Deserialize 中：
ComponentSerializer.SerializeComponent(ref comp, writer);
```

删除所有手写的 `WriteXxxComponent` / `ReadXxxComponent` 方法。

---

## 5. 集成到 PrefabInstantiator

**文件**: `Public/Prefab/PrefabInstantiator.cs`

替换 `AddComponentFromData` 中的硬编码 switch：

```csharp
private void AddComponentFromData(IEntity entity, PrefabComponentData compData)
{
    using var doc = JsonDocument.Parse(compData.Data);
    var entity2 = ((FrifloEntity)entity).InternalEntity;
    ComponentSerializer.TryAddComponent(entity2, compData.TypeName, doc.RootElement);
}
```
