## 1. 系统总体架构

### 1.1 Launcher 在引擎体系中的位置

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Neverness Engine 体系                        │
│                                                                     │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐          │
│  │   Launcher   │    │    Editor    │    │   Runtime    │          │
│  │  (Avalonia)  │───▶│  (C# ImGui)  │───▶│   (C++ DLL)  │          │
│  │  .NET 9      │    │  .NET 9      │    │  Native      │          │
│  └──────────────┘    └──────────────┘    └──────────────┘          │
│         │                    │                    │                 │
│         ▼                    ▼                    ▼                 │
│  ┌─────────────────────────────────────────────────────────────┐   │

│  │  ├── Bin/          (编译产物)                                │   │

│  │  ├── Tools/        (工具链)                                  │   │
│  │  └── Config/       (引擎配置)                                │   │

│  │  ├── Scenes/      (场景文件)                                 │   │
│  │  └── Config/      (项目配置)                                 │   │
│  └─────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 与 Editor / Runtime / Project 的关系

| 组件 | Launcher 的关系 | 交互方式 |
|------|-----------------|----------|
| **Editor** | Launcher 启动 Editor 进程 | 传递项目路径、引擎路径、启动参数 |
| **Runtime** | Launcher 启动 Game 进程 | 传递项目路径、启动参数、Debug 标志 |
| **Project** | Launcher 扫描并管理项目 | 读取 `.nnproject` 文件，展示项目列表 |
| **Engine Root** | Launcher 定位引擎安装 | 扫描注册表、环境变量、固定路径 |

### 1.3 核心职责边界

| 职责 | 说明 | 是否核心 |
|------|------|----------|
| 启动 Editor | 根据项目配置启动对应版本 Editor | ✅ 核心 |
| 启动 Runtime/Game | 根据项目配置启动 Runtime 或 Game | ✅ 核心 |
| 管理引擎路径 | 扫描本地引擎安装，支持多版本 | ✅ 核心 |
| 管理项目 | 扫描项目、最近项目列表、项目创建 | ✅ 核心 |
| 读取 `.nnproject` | 解析项目配置，绑定引擎版本 | ✅ 核心 |
| Debug/Development 参数 | 传递调试启动参数 | ✅ 核心 |
| 日志与崩溃记录 | 记录启动失败、崩溃信息 | ✅ 核心 |
| 项目创建向导 | 引导用户创建新项目 | 🔜 近期 |
| 引擎下载管理 | 下载/安装引擎版本 | 🔜 近期 |
| 插件系统 | Launcher 自身插件扩展 | 🔮 未来 |
| CLI 启动支持 | 命令行启动 Editor/Game | 🔮 未来 |

### 1.4 明确禁止设计

以下内容不在 Launcher 职责范围内，不得设计：

- ❌ 游戏商店 / 内容分发
- ❌ DLC / 付费内容管理
- ❌ 账号系统 / 登录认证
- ❌ CDN 分发平台
- ❌ 在线社区 / 社交功能
- ❌ 游戏内购买
- ❌ 成就系统
- ❌ 云存档同步（Launcher 级别）
- ❌ Steam/Epic 级别的游戏库管理

### 2.2 各层职责说明

| 层级 | 目录 | 职责 | 依赖方向 |
|------|------|------|----------|
| **Core** | `Core/` | 领域模型、枚举、常量，零外部依赖 | 无依赖 |
| **Contracts** | `Contracts/` | 接口定义，服务契约 | → Core |
| **Services** | `Services/` | 业务逻辑实现 | → Core, Contracts |
| **Infrastructure** | `Infrastructure/` | 外部系统适配（进程、文件、JSON） | → Core, Contracts |
| **ViewModels** | `ViewModels/` | MVVM ViewModel，UI 状态管理 | → Core, Contracts |
| **Views** | `Views/` | Avalonia XAML 视图 | → ViewModels |
| **UI** | `Converters/`, `Controls/`, `Themes/`, `Resources/` | UI 基础设施 | → Avalonia |

### 2.3 设计原则

- **单向依赖**：上层依赖下层，禁止反向
- **接口隔离**：Services 通过 Contracts 接口暴露
- **最小依赖**：Core 层零外部依赖，仅使用 BCL
- **工具优先**：每个模块服务于"启动 Editor/Game"这一核心目标

## 3. 启动流程设计 ✅

### 3.1 Launcher 启动流程

```
程序启动 (Program.Main)
    │
    ▼
构建 IHost (Microsoft.Extensions.Hosting)
    │  ├── 注册所有服务到 DI 容器
    │  ├── 配置日志
    │  └── 配置选项
    │
    ▼
IHost.StartAsync()
    │  ├── IConfigurationService.Load()
    │  │   ├── 加载 LauncherConfig（安装目录配置）
    │  │   └── 加载 UserSettings（用户偏好）
    │  │
    │  ├── ILogService.Initialize()
    │  │   └── 初始化日志文件
    │  │
    │  └── 等待所有服务就绪
    │
    ▼

    │  ├── AppBuilder.Configure<App>()
    │  └── 传入 IServiceProvider
    │
    ▼
App.OnFrameworkInitializationCompleted()
    │  ├── 从 DI 获取 MainWindowViewModel
    │  ├── 注册页面到 NavigationService
    │  └── 创建 MainWindow
    │
    ▼

    │
    ├── IEngineService.ScanEngines()
    │   ├── 扫描注册表 (HKLM/HKCU\Software\Neverness)
    │   ├── 扫描环境变量 (NEVERNESS_ENGINE_ROOT)
    │   ├── 扫描固定路径 (C:\Neverness\Engine)
    │   └── 扫描自定义路径（用户配置）
    │
    ├── IProjectService.ScanProjects()
    │   ├── 从最近项目列表加载
    │   ├── 从配置的项目目录扫描

    │  ├── 展示可用引擎版本

    │
    ▼
Launcher 就绪，等待用户操作
```

### 3.2 Engine Root 解析流程

```
IEngineService.ScanEngines()
    │
    ▼
扫描策略（按优先级）
    │
    ├── 1. 环境变量扫描
    │   └── 检查 NEVERNESS_ENGINE_ROOT 环境变量
    │       ├── 存在且有效 → 添加到引擎列表
    │       └── 不存在 → 跳过
    │
    ├── 2. 注册表扫描 (Windows)
    │   ├── HKLM\Software\Neverness\Engine\Installations
    │   └── HKCU\Software\Neverness\Engine\Installations
    │       └── 遍历每个注册表项，读取路径和版本
    │
    ├── 3. 固定路径扫描
    │   ├── C:\Neverness\Engine
    │   ├── D:\Neverness\Engine
    │   └── 用户自定义扫描路径
    │
    └── 4. 相对路径扫描
        └── Launcher 安装目录的上级目录
    │
    ▼
对每个候选路径验证
    │
    ├── 检查目录是否存在
    ├── 检查是否包含引擎标识文件 (EngineVersion.json)
    ├── 解析引擎版本号
    ├── 验证引擎完整性（关键文件是否存在）
    └── 创建 EngineVersion 对象
    │
    ▼
返回引擎列表，按版本号降序排列
```

### 3.3 Project Scan 流程

```
IProjectService.ScanProjects()
    │
    ▼
扫描来源
    │
    ├── 1. 最近项目列表
    │   └── IRecentProjectService.GetRecentProjects()
    │       ├── 加载最近项目缓存
    │       ├── 验证每个项目路径是否有效
    │       └── 移除无效项目
    │
    ├── 2. 配置的项目目录
    │   └── UserSettings.ProjectDirectories[]
    │       ├── 遍历每个目录
    │       ├── 递归查找 .nnproject 文件
    │       └── 解析项目信息
    │
    └── 3. 用户手动打开
        └── 用户通过文件对话框选择 .nnproject 文件
    │
    ▼
对每个发现的项目
    │
    ├── 读取 .nnproject 文件
    ├── 解析 JSON 为 ProjectInfo
    ├── 验证项目路径有效性
    ├── 检查关联引擎版本是否可用
    └── 创建 ProjectCardViewModel
    │
    ▼
返回项目列表，按最近修改时间排序
```

### 3.4 Editor 启动流程

```
用户选择项目 → 点击 "打开 Editor"
    │
    ▼
MainWindowViewModel 确认选中项目
    │
    ▼
ILaunchService.LaunchEditor(projectInfo, launchOptions)
    │
    ├── 解析引擎路径
    │   ├── 从 ProjectInfo.EngineVersion 获取版本要求
    │   ├── IEngineService.FindEngine(version)
    │   │   ├── 在已扫描的引擎列表中查找匹配版本
    │   │   └── 找到 → 返回 EngineVersion 对象
    │   │   └── 未找到 → 提示用户下载或手动指定
    │   └── 获取引擎可执行文件路径
    │
    ├── 构建启动参数
    │   ├── 项目路径: --project "C:\Projects\MyGame"
    │   ├── 引擎路径: --engine "C:\Neverness\Engine\5.0"
    │   ├── 启动模式: --mode editor
    │   ├── Debug 参数: --debug (如果启用)
    │   └── 额外参数: 用户自定义
    │
    ├── 验证可执行文件存在
    │   └── Editor.exe 路径 = EngineRoot/Bin/Editor/NevernessEditor.exe
    │
    └── 启动进程
        ├── ProcessLauncher.Launch(editorPath, arguments)
        ├── 记录进程 PID
        ├── 注册进程退出事件
        └── 返回 ProcessHandle
    │
    ▼
更新 UI 状态
    │  ├── 项目卡片显示 "Editor 运行中"
    │  └── 记录到最近项目
    │
    ▼
IRecentProjectService.RecordAccess(projectPath)
    │  └── 更新最近项目访问时间
```

### 3.5 Game 启动流程

```
用户选择项目 → 点击 "启动 Game"
    │
    ▼
ILaunchService.LaunchGame(projectInfo, launchOptions)
    │
    ├── 解析引擎路径（同 Editor）
    │
    ├── 构建启动参数
    │   ├── 项目路径: --project "C:\Projects\MyGame"
    │   ├── 启动模式: --mode game
    │   ├── 窗口模式: --windowed / --fullscreen
    │   ├── 分辨率: --width 1920 --height 1080
    │   ├── Debug 参数: --debug (如果启用)
    │   └── 额外参数: 用户自定义
    │
    ├── 验证可执行文件存在
    │   └── Runtime.exe 路径 = EngineRoot/Bin/Runtime/NevernessRuntime.exe
    │
    └── 启动进程
        ├── ProcessLauncher.Launch(runtimePath, arguments)
        ├── 记录进程 PID
        └── 返回 ProcessHandle
    │
    ▼
Game 运行期间
    │  ├── 监控进程状态
    │  └── 捕获崩溃信息
    │
    ▼
Game 退出
    │  ├── 记录退出码
    │  ├── 如果崩溃 → 保存崩溃日志
    │  └── 更新 UI 状态
```

### 11.2 Service 注册策略

| 服务 | 生命周期 | 理由 |
|------|----------|------|
| IProjectService | Singleton | 全局共享项目列表 |
| IEngineService | Singleton | 全局共享引擎版本列表 |
| ILaunchService | Singleton | 管理所有运行中的进程 |
| IRecentProjectService | Singleton | 全局共享最近项目列表 |

| IFileSystem | Singleton | 无状态，可共享 |
| IProcessLauncher | Singleton | 无状态，可共享 |
| ILogService | Singleton | 全局日志 |
| MainWindowViewModel | Singleton | 主窗口唯一 |
| PageViewModels | Transient | 每次导航创建新实例 |
| ProjectCardViewModel | Transient | 每个项目卡片独立 |

### 11.3 Singleton 使用原则

- 服务是否有全局状态？是 → Singleton
- 服务是否需要跨页面共享？是 → Singleton
- 服务是否管理外部资源（进程、文件）？是 → Singleton

### 11.4 Transient 使用原则

- ViewModel 是否每次使用都是独立的？是 → Transient
- ViewModel 是否需要绑定到独立的 UI 元素？是 → Transient

## 14. 关键设计原则总结

### 14.1 工具优先，不做平台

| 原则 | 说明 |
|------|------|
| 职责边界清晰 | Launcher 只负责启动和项目管理，不参与引擎运行逻辑 |
| 不做内容分发 | 不设计商店、DLC、付费内容等平台功能 |
| 不做用户系统 | 不设计账号、登录、社交等平台功能 |
| 保持轻量 | 每个功能都服务于"启动 Editor/Game"这一核心目标 |

### 14.2 不影响 Editor / Runtime

| 原则 | 说明 |
|------|------|
| 进程隔离 | Launcher 启动 Editor/Game 后，两者独立运行 |
| 参数传递 | 通过命令行参数传递配置，不修改引擎文件 |
| 崩溃隔离 | Editor/Game 崩溃不影响 Launcher |
| 版本独立 | Launcher 版本与引擎版本解耦 |

### 14.3 启动速度优先

| 原则 | 说明 |
|------|------|
| 最小化启动依赖 | 启动时不加载不必要的服务 |
| 异步初始化 | 所有 I/O 操作异步执行 |
| 延迟加载 | 项目详情、引擎信息等按需加载 |
| 缓存策略 | 最近项目、引擎列表等缓存到内存 |

### 14.4 可维护性优先

| 原则 | 说明 |
|------|------|
| 清晰分层 | Core/Contracts/Services/Infrastructure/ViewModels/Views |
| 接口隔离 | 通过接口解耦，便于测试和替换 |
| 单一职责 | 每个类/服务只负责一件事 |
| 依赖倒置 | 上层依赖抽象，下层实现抽象 |

### 14.5 AI 可读结构优先

| 原则 | 说明 |
|------|------|
| 一致的命名 | 类名、方法名、属性名遵循一致的命名规范 |
| 清晰的目录结构 | 每个目录有明确的职责 |
| 完整的文档 | 每个模块有清晰的说明 |
| 可预测的代码 | 相似功能使用相似的实现模式 |
