# Play Mode 快照恢复失败修复

> 日期：2026-05-30
> 模块：Runtime VFS / Editor PlayMode
> 严重程度：高（播放模式退出后场景数据未恢复）

---

## 问题描述

编辑器进入播放模式后，对 Scene 和 Component 的修改在退出播放模式后仍然保留，快照恢复机制完全失效。

## 根因分析

问题出在 C++ VFS 层的 `MemoryFileSystem`（`Engine/Source/Runtime/NNRuntimeVFS/Include/VFS/MemoryFileSystem.h`），有两个 bug 共同导致快照数据丢失。

### Bug 1: `OpenFileST` 无条件创建空文件

```cpp
// 原代码（第 271-291 行）
inline IFilePtr OpenFileST(const FileInfo& filePath, IFile::FileMode mode)
{
    IFilePtr file = FindFile(filePath, m_FileList);
    bool isExists = (file != nullptr);
    if (!isExists && !IsReadOnlyST()) {
        file.reset(new MemoryFile(filePath));
    }

    if (file) {
        // *** BUG: 无条件创建全新空文件，丢弃了缓存中包含数据的文件 ***
        file.reset(new MemoryFile(filePath));

        file->Open(mode);

        if (!isExists && file->IsOpened()) {
            m_FileList[filePath.AbsolutePath()] = file;
        }
    }

    return file;
}
```

- 步骤 (A)：`FindFile` 成功从 `m_FileList` 中找到了之前写入时缓存的 MemoryFile 对象（包含快照数据）。
- 步骤 (C)：**无条件** `file.reset(new MemoryFile(filePath))` 丢弃了对缓存对象的引用，创建了一个全新的空 MemoryFile。
- 由于 `m_FileList` 中仍持有对旧对象的 shared_ptr，旧对象不会被销毁（不泄漏），但**新创建的空文件被返回给调用方**。

### Bug 2: `CloseFileST` 在 Close 时清除缓存

```cpp
// 原代码（第 293-301 行）
inline void CloseFileST(IFilePtr file)
{
    if (!file) {
        return;
    }

    file->Close();
    m_FileList.erase(file->GetFileInfo().AbsolutePath());  // *** BUG: 数据丢失 ***
}
```

写入完成后 `Close()` 会把文件从 `m_FileList` 缓存中移除。下次读取时 `FindFile` 找不到文件，创建空文件，读到 0 字节。

### 完整失败链

```
CaptureSnapshot
  → SerializeScene → WriteTextToFile
    → OpenFile(Write) → 创建 MemoryFile → 写入 JSON 数据 → Close()
    → CloseFileST() → m_FileList.erase() → 文件从缓存中被删除

RestoreSnapshot
  → DeserializeScene → ReadTextFromFile
    → OpenFile(Read) → FindFile() 找不到（已被 erase）
    → 创建空 MemoryFile → 读取 0 字节 → 反序列化失败
    → ReplaceActiveSceneFromSnapshot 返回 Invalid
    → 旧的（已修改的）场景保持不变
```

### 附加问题：Editor 侧缓存未刷新

`SceneHierarchyCache` 不感知 sceneHandle 变化——版本号比较可能误判为"未变"，导致实体列表不刷新。`DetailInspector` 的选中状态指向旧场景的实体，从未被清除。

### Fix 1: `MemoryFileSystem::OpenFileST`（C++ VFS 层）

移除无条件 `file.reset()`，仅在写入+截断模式下创建新文件替换缓存：

```cpp
inline IFilePtr OpenFileST(const FileInfo& filePath, IFile::FileMode mode)
{
    IFilePtr file = FindFile(filePath, m_FileList);
    bool isExists = (file != nullptr);

    // 文件不存在且文件系统可写 → 创建新文件
    if (!isExists && !IsReadOnlyST()) {
        file.reset(new MemoryFile(filePath));
    }

    if (file) {
        // 写入模式：需要截断已有文件数据时，创建全新文件替换缓存
        if (isExists && (mode & IFile::FileMode::Truncate) == IFile::FileMode::Truncate) {
            file.reset(new MemoryFile(filePath));
            isExists = false; // 标记为新文件，需要写入缓存
        }

        file->Open(mode);

        // 新文件打开成功 → 写入缓存（保留数据供后续读取）
        if (!isExists && file->IsOpened()) {
            m_FileList[filePath.AbsolutePath()] = file;
        }
    }

    return file;
}
```

### Fix 2: `MemoryFileSystem::CloseFileST`（C++ VFS 层）

Close 时不再从缓存中移除文件，数据保留在 `m_FileList` 中供后续读取：

```cpp
inline void CloseFileST(IFilePtr file)
{
    if (!file) {
        return;
    }

    file->Close();
    // 注意：不从 m_FileList 中移除文件。
    // 文件数据需要保留在缓存中，供后续 Open(Read) 读取。
    // 显式删除文件应使用 RemoveFileST。
}
```

### Fix 3: `SceneHierarchyCache.TryRefresh`（C# Editor 层）

增加 sceneHandle 感知，场景切换时强制全量刷新 + 清除旧 UI 状态：

```csharp
private ulong _cachedSceneHandle;

public unsafe bool TryRefresh(ulong sceneHandle)
{
    // 场景句柄变化 → 强制全量刷新 + 清除旧 UI 状态
    if (sceneHandle != _cachedSceneHandle)
    {
        _cachedSceneHandle = sceneHandle;
        _cachedVersion = ulong.MaxValue;
        _selectedSet.Clear();
        _expandedSet.Clear();
        _searchText = "";
        _visibleDirty = true;
    }
    // ... 原有版本轮询逻辑
}
```

### Fix 4: `DetailInspector.ClearSelection`（C# Editor 层）

新增 `ClearSelection()` 方法，场景切换时清除选中状态：

```csharp
internal void ClearSelection()
{
    _selectedEntityHandle = 0;
    _selectedEntityName = "";
}
```

### Fix 5: `SceneModuleImp.SetSceneHandle`（C# Editor 层）

场景切换时调用 `ClearSelection()`：

```csharp
public static void SetSceneHandle(ulong sceneHandle)
{
    if (s_sceneBrowser != null)
        s_sceneBrowser.SceneHandle = sceneHandle;
    if (s_detailInspector != null)
    {
        s_detailInspector.SceneHandle = sceneHandle;
        s_detailInspector.ClearSelection();
    }
    if (s_editorViewport != null)
        s_editorViewport.SetScene(sceneHandle);
}
```
