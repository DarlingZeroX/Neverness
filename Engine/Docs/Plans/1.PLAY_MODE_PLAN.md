# Scene 播放模式（Play Mode）框架计划

> Neverness 第一代 Play Mode 架构——基于工业级编辑器经验（Unity / UE / Godot）实施版。

## 背景

编辑器工具栏已有 Play/Stop 按钮 UI，但完全 stub。`EditorState.PlayMode` 枚举和 `EditorEventType.PlayModeChanged` 事件类型已定义但从未被使用。需要实现完整的进入/退出播放模式流程。

## 现有基础设施盘点

| 已有 | 说明 |
|------|------|
| `PlayMode` 枚举 (Editing/Playing/Paused) | `Editor.Core.Public.EditorState` |
| `EditorState.PlayMode` / `IsPlaying` | 全局状态属性，但 setter 从未被调用 |
| `EditorEventType.PlayModeChanged` | 事件类型已定义，但从未被正确 emit |
| `HotReloadSnapshot` 机制 | `SceneWorld.SaveSnapshot()` / `RestoreFromSnapshot()` — 仅保存 Managed 侧映射 |
| `SceneNativeBridge.SerializeScene` / `DeserializeScene` | 序列化/反序列化到 VFS 路径（VGSC 二进制格式） |
| `SceneSystemScheduler` | 已支持全 TickGroup，但无法按标签过滤 |
| `SceneSubsystem` | 注册到 RuntimeLoop，驱动 `SceneManager.TickActiveScene()` |

---

## 核心设计原则

### 原则 1: PlayMode 是 Editor 概念，不属于 Runtime

Runtime.Scene 不知道 PlayMode / Editor / Pause 的存在。它只提供 `SceneSystemTags` 标签机制和 `Tick(tagMask)` 过滤能力。什么时候 tick、tick 哪些标签，由 Editor 侧的 `PlayModeController` 决定。

### 原则 2: SceneSystemTags 只定义原子标签

Runtime 定义 `SceneSystemTags`（Flags 枚举），只含原子标签。组合标签（EditingMask / PlayingMask / PausedMask）由 Editor 侧 `PlayModeController` 负责。

### 原则 3: 默认值通过常量类定义

未标记 `[SceneSystemTag]` 的系统默认 `Render | Audio | Streaming`。通过 `SceneSystemTagDefaults.Default` 常量统一引用，不使用已删除的 `SceneSystemTags.Always`。

### 原则 4: 播放期间允许全部编辑

Unity 风格：播放时可以修改 Transform、创建/删除 Entity。Stop 后快照恢复，一切回到原样。

### 原则 5: EditorState 不保存备份

`EditorState` 只持有当前状态。Pre-play 快照放在 `PlayModeController` 内部的 `PlayModeSnapshot` 中。

### 原则 6: Pause 不冻结 Always 系统

```
Editing  → Editor | Render | Audio | Streaming
Playing  → Gameplay | Physics | Render | Audio | Streaming
Paused   → Render | Audio | Streaming（Gameplay 和 Editor 都暂停）
```

### 原则 7: Native Tick 必须同步过滤

`SceneNativeBridge.TickSystems()` 接受 `tagMask` 参数，Native 侧同样按模式过滤。否则暂停时 Native Gameplay 继续跑（AI 移动、物理模拟），Pause 形同虚设。

### 原则 8: RuntimeLoop 统一驱动

不要绕过 `RuntimeMainLoop.Tick()`。`SceneSubsystem` 保持在 RuntimeLoop 中，内部委托给 `PlayModeController.TickActiveScene()`。所有 Subsystem 走统一 Tick 链，Profiling 简单。

### 原则 9: 快照用 Guid.NewGuid() 命名

快照路径用 `Guid.NewGuid()` 生成，避免多次 Play/Stop 或多实例冲突。`PlayModeSnapshot` 同时保存 `SceneGuid` 用于恢复索引。

### 原则 10: RestoreSnapshot 用原子操作，不触发事件风暴

使用 `SceneManager.ReplaceActiveSceneFromSnapshot()` 替代 `Unload + Load` 组合。避免触发 `SceneUnloaded` / `SceneLoaded` / `SceneActivated` 事件风暴，也不会打断未来 Multi-Scene / Streaming 场景中其他子场景的状态。

### 原则 11: EnterPlay 必须异常安全

`CaptureSnapshot()` 可能失败（磁盘满、权限问题、VFS 异常）。如果快照保存失败，不能切换 PlayMode 状态。使用 `try/catch` + 返回值保证原子性。

### 原则 12: Deferred Queue 按标签 Flush

`TickByTagMask()` 结束时调用 `Events.FlushDeferred(mask)` 而非 `Events.FlushDeferred()`。Pause 时只 Flush Always 标签的队列，不 Flush Gameplay 队列。Phase 1 实现基础版（全 Flush），但接口预留为 `FlushDeferred(mask)`。

```csharp
/// <summary>
/// 场景系统标签——标记系统的执行类别。
/// Runtime 侧仅提供标签机制，不知道 PlayMode 的存在。
/// Editor 侧决定每个模式用什么 tagMask。
///
/// 只定义原子标签，不定义组合标签。
/// 组合逻辑由 Editor 侧 PlayModeController 负责。
/// </summary>
[Flags]
public enum SceneSystemTags : ushort
{
    None        = 0,

    /// <summary>编辑器系统（Gizmo、选择高亮、层级同步）。</summary>
    Editor      = 1 << 0,

    /// <summary>游戏逻辑系统（角色控制器、AI、游戏状态机）。</summary>
    Gameplay    = 1 << 1,

    /// <summary>渲染系统（Render、PostProcess）。</summary>
    Render      = 1 << 2,

    /// <summary>音频系统（Audio、Music）。</summary>
    Audio       = 1 << 3,

    /// <summary>物理系统（Physics、Collision）。</summary>
    Physics     = 1 << 4,

    /// <summary>流式加载系统（Streaming、LOD）。</summary>
    Streaming   = 1 << 5,
}

/// <summary>
/// SceneSystemTags 默认值常量。
/// 未标记 [SceneSystemTag] 的系统默认为此值。
/// </summary>
internal static class SceneSystemTagDefaults
{
    /// <summary>
    /// 默认标签：Render | Audio | Streaming。
    /// 所有模式都应 tick 的基础设施系统。
    /// </summary>
    public const SceneSystemTags Default =
        SceneSystemTags.Render |
        SceneSystemTags.Audio |
        SceneSystemTags.Streaming;
}

/// <summary>
/// 标记 ECS 系统的标签。
/// 未标记的系统默认为 <see cref="SceneSystemTagDefaults.Default"/>。
/// </summary>
[AttributeUsage(AttributeTargets.Class, Inherited = false)]
public sealed class SceneSystemTagAttribute(SceneSystemTags tags) : Attribute
{
    public SceneSystemTags Tags { get; } = tags;
}
```

#### Step 6: SceneManager 新增 ReplaceActiveSceneFromSnapshot

**修改文件**: `Runtime.Scene/Public/SceneManager.cs`

新增原子快照恢复 API，替代 `Unload + Load` 组合：

```csharp
/// <summary>
/// 从快照原子替换当前激活场景。
/// 不触发 SceneUnloaded / SceneLoaded / SceneActivated 事件风暴。
/// 内部：dispose old → deserialize new → 替换 _activeWorld 引用。
/// </summary>
public NNSceneResult ReplaceActiveSceneFromSnapshot(NNGuid sceneGuid, string snapshotPath)
{
    if (_activeWorld == null)
        return NNSceneResult.Invalid;

    // 1. 反序列化快照
    var result = SceneNativeBridge.DeserializeScene(out var newHandle, snapshotPath);
    if (result != NNSceneResult.Ok || newHandle == 0)
        return NNSceneResult.Invalid;

    // 2. 保存旧 world 引用
    var oldWorld = _activeWorld;
    var oldName = oldWorld.Name;

    // 3. Dispose 旧 world（释放 Native 资源 + Managed 映射）
    //    但不触发 SceneUnloaded 事件（这是快照恢复，不是正常卸载）
    oldWorld.DisposeQuiet();  // 新增方法：静默 Dispose，不触发事件

    // 4. 创建新 world 并替换
    var newWorld = new SceneWorld(newHandle, oldName);
    newWorld.AssetGuid = sceneGuid;
    // 注意：不设置 AssetPath（快照路径不是正式资产路径）

    _worlds[oldName] = newWorld;
    _activeWorld = newWorld;

    // 5. 触发一次 SceneActivated 事件（通知 Editor 面板刷新）
    SceneActivated?.Invoke(newWorld);

    return NNSceneResult.Ok;
}
```

> **与 `Unload + Load` 的关键区别**：
> - 不触发 `SceneUnloaded` 事件（避免其他系统误清理状态）
> - 不触发 `SceneLoaded` 事件（快照恢复不是新场景加载）
> - 只触发一次 `SceneActivated`（通知 UI 刷新）
> - 不影响 `_worlds` 字典结构（保持 world 名称不变）
> - 未来 Multi-Scene 场景下，只替换 active world，不影响其他子场景

```csharp
/// <summary>
/// 播放模式生命周期事件类型。
/// 细粒度事件，供插件（Profiler、Audio Preview、Animation Preview）挂载。
/// </summary>
public enum PlayModeEventType
{
    EnteringPlayMode,   // 即将进入播放（快照已保存，尚未开始 tick）
    EnteredPlayMode,    // 已进入播放（Gameplay 系统开始 tick）
    ExitingPlayMode,    // 即将退出播放（即将恢复快照）
    ExitedPlayMode,     // 已退出播放（快照已恢复，回到编辑态）
    Paused,             // 已暂停（Always 系统继续，Gameplay/Editor 暂停）
    Resumed,            // 已恢复（从暂停恢复到播放）
}

/// <summary>播放模式生命周期事件数据。</summary>
public readonly record struct PlayModeEvent(PlayModeEventType Type);

/// <summary>
/// PlayModeChanged 事件 payload——类型化，避免字符串拼写错误。
/// 通过 EditorEventBus 分发，挂在 EditorEventType.PlayModeChanged 上。
/// </summary>
public readonly record struct PlayModeChangedEvent(
    PlayMode OldMode,
    PlayMode NewMode);
```

#### Step 9: PlayModeController

**新增文件**: `Editor.Scene/Private/PlayMode/PlayModeController.cs`

```csharp
/// <summary>
/// 播放模式控制器——Editor 侧核心，驱动 PlayMode 状态机。
/// Runtime.Scene 完全不知道 PlayMode 的存在。
/// </summary>
internal sealed class PlayModeController
{
    private readonly SceneManager _sceneManager;
    private readonly EditorState _editorState;
    private readonly IEditorEventBus _eventBus;

    private PlayModeSnapshot? _prePlaySnapshot;
    private PlayMode _currentMode = PlayMode.Editing;

    // ── Editor 决定每个模式的 tagMask（组合标签在此定义，不放 Runtime）──

    private static readonly SceneSystemTags s_editingMask =
        SceneSystemTags.Editor |
        SceneSystemTags.Render |
        SceneSystemTags.Audio |
        SceneSystemTags.Streaming;

    private static readonly SceneSystemTags s_playingMask =
        SceneSystemTags.Gameplay |
        SceneSystemTags.Physics |
        SceneSystemTags.Render |
        SceneSystemTags.Audio |
        SceneSystemTags.Streaming;

    private static readonly SceneSystemTags s_pausedMask =
        SceneSystemTags.Render |
        SceneSystemTags.Audio |
        SceneSystemTags.Streaming;

    // ── 公开状态 ──

    public PlayMode CurrentMode => _currentMode;
    public bool IsPlaying => _currentMode == PlayMode.Playing;
    public bool IsPaused => _currentMode == PlayMode.Paused;

    // ── 生命周期（异常安全）──

    /// <summary>
    /// 进入播放模式。
    /// 快照保存失败时不会切换状态，保证原子性。
    /// </summary>
    public bool EnterPlay()
    {
        if (_currentMode != PlayMode.Editing) return false;
        if (!_sceneManager.HasActiveScene) return false;

        var oldMode = _currentMode;

        _eventBus.Emit(new PlayModeEvent(PlayModeEventType.EnteringPlayMode));

        // 快照可能失败（磁盘满、权限、VFS 异常），必须异常安全
        try
        {
            _prePlaySnapshot = CaptureSnapshot();
        }
        catch (Exception ex)
        {
            // 快照失败，不切换状态
            Console.Error.WriteLine($"[PlayModeController] 快照保存失败: {ex.Message}");
            return false;
        }

        _currentMode = PlayMode.Playing;
        _editorState.PlayMode = PlayMode.Playing;

        _eventBus.Emit(new PlayModeEvent(PlayModeEventType.EnteredPlayMode));
        _eventBus.Emit(new EditorEvent(EditorEventType.PlayModeChanged,
            new PlayModeChangedEvent(oldMode, PlayMode.Playing)));

        return true;
    }

    /// <summary>
    /// 退出播放模式——恢复快照，回到编辑态。
    /// </summary>
    public bool ExitPlay()
    {
        if (_currentMode != PlayMode.Playing && _currentMode != PlayMode.Paused)
            return false;

        var oldMode = _currentMode;

        _eventBus.Emit(new PlayModeEvent(PlayModeEventType.ExitingPlayMode));

        RestoreSnapshot();

        _currentMode = PlayMode.Editing;
        _editorState.PlayMode = PlayMode.Editing;

        _eventBus.Emit(new PlayModeEvent(PlayModeEventType.ExitedPlayMode));
        _eventBus.Emit(new EditorEvent(EditorEventType.PlayModeChanged,
            new PlayModeChangedEvent(oldMode, PlayMode.Editing)));

        CleanupSnapshot();

        return true;
    }

    public bool Pause()
    {
        if (_currentMode != PlayMode.Playing) return false;

        var oldMode = _currentMode;

        _currentMode = PlayMode.Paused;
        _editorState.PlayMode = PlayMode.Paused;

        _eventBus.Emit(new PlayModeEvent(PlayModeEventType.Paused));
        _eventBus.Emit(new EditorEvent(EditorEventType.PlayModeChanged,
            new PlayModeChangedEvent(oldMode, PlayMode.Paused)));

        return true;
    }

    public bool Resume()
    {
        if (_currentMode != PlayMode.Paused) return false;

        var oldMode = _currentMode;

        _currentMode = PlayMode.Playing;
        _editorState.PlayMode = PlayMode.Playing;

        _eventBus.Emit(new PlayModeEvent(PlayModeEventType.Resumed));
        _eventBus.Emit(new EditorEvent(EditorEventType.PlayModeChanged,
            new PlayModeChangedEvent(oldMode, PlayMode.Playing)));

        return true;
    }

    /// <summary>单步执行——暂停状态下执行一帧后继续暂停。Phase 1 预留接口。</summary>
    public bool StepFrame()
    {
        // TODO: Phase 2 实现
        throw new NotImplementedException("Frame Step 将在 Phase 2 实现");
    }

    // ── Tick 驱动（由 SceneSubsystem 委托调用）──

    /// <summary>
    /// 每帧调用——根据当前模式选择 tagMask，驱动 SceneWorld tick。
    /// 由 SceneSubsystem.Tick() 委托调用，保持 RuntimeLoop 统一驱动。
    /// </summary>
    public void TickActiveScene(float deltaTime)
    {
        var world = _sceneManager.ActiveWorld;
        if (world == null || !world.IsValid) return;

        var mask = _currentMode switch
        {
            PlayMode.Editing => s_editingMask,
            PlayMode.Playing => s_playingMask,
            PlayMode.Paused  => s_pausedMask,
            _ => s_editingMask,
        };

        world.TickByTagMask(deltaTime, mask);
    }

    // ── 快照内部实现 ──

    private PlayModeSnapshot CaptureSnapshot()
    {
        var world = _sceneManager.ActiveWorld!;

        // 用 Guid.NewGuid() 而非 sceneGuid，避免多次 Play/Stop 或多实例冲突
        var snapshotPath = $"Cache/PlayModeSnapshots/{Guid.NewGuid():N}.vgsc";

        world.Save(snapshotPath);

        return new PlayModeSnapshot
        {
            SnapshotPath = snapshotPath,
            SceneGuid = world.AssetGuid,
            SceneAssetPath = world.AssetPath,
            SelectedEntityHandle = _editorState.SelectedEntityHandle,
            CapturedAt = DateTimeOffset.UtcNow,
        };
    }

    private void RestoreSnapshot()
    {
        if (_prePlaySnapshot == null) return;

        // 原子替换——不触发事件风暴，不影响其他子场景
        _sceneManager.ReplaceActiveSceneFromSnapshot(
            _prePlaySnapshot.SceneGuid,
            _prePlaySnapshot.SnapshotPath);

        _editorState.SelectedEntityHandle = _prePlaySnapshot.SelectedEntityHandle;
    }

    private void CleanupSnapshot()
    {
        if (_prePlaySnapshot == null) return;

        try
        {
            // VFS.Delete(_prePlaySnapshot.SnapshotPath);
        }
        catch { }

        _prePlaySnapshot = null;
    }
}
```
