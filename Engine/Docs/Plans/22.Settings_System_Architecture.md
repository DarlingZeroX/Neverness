> 日期：2026-06-23

> 状态：Phase 1-2.9 已完成，Phase 3 待做

## 一、需求概述

| 需求 | 说明 |
|------|------|
| Runtime Project Setting | 游戏运行时设置（分辨率、音频、输入等），存储在项目目录 |
| Editor Preferences | 编辑器偏好设置（IDE、主题、语言等），存储在用户目录 |
| 设置范围分离 | Project Settings 和 Preferences 使用独立窗口，类似 Unity |
| UI 渲染自动化 | 简单字段自动生成控件，复杂页面手写 AXAML |
| 保存/读取自动化 | JSON 序列化，AOT 兼容（Phase 1 不用 Source Generator） |
| 外部扩展 | 其他模块通过 Attribute/API 注册自己的设置表 |
| AOT 兼容 | System.Text.Json Source Generator（JsonSerializerContext） |

---

## 二、核心设计决策

### 决策 1：Runtime.Settings 零引擎依赖

```
Neverness.Runtime.Settings
├─ System.Text.Json（唯一外部依赖）
└─ BCL
```

不依赖 VFS、ProjectSystem、AssetSystem。路径由调用者传入。
好处：Launcher、Server、Unit Test 都能直接用。

### 决策 2：Phase 1 不用 Source Generator

先手写 Descriptor，验证架构稳定后再加 Generator。
原因：Descriptor 模型还未稳定，过早引入 Generator 会放大修改成本。

### 决策 3：混合 UI 模式

```
简单页面 → 自动生成（AutoSettingsPanel）
复杂页面 → 手写 AXAML + 复用 AutoSettingsField 控件
自定义 → [CustomEditor(typeof(...))] 走专用编辑器
```

### 决策 4：Settings 与 Inspector 共享 Attribute 体系

`[DisplayName]`、`[Range]`、`[Hidden]` 等 Attribute 定义在 Runtime.Settings，
Inspector 侧的 `[HideInInspector]`、`[Range]` 可逐步对齐或共用。

### 决策 5：Project Settings 与 Preferences 分离

类似 Unity 的设计，使用 `SettingsScope` 枚举区分两种设置范围：
- **Project**：项目级设置，随项目存储，团队共享（图形、音频等）
- **User**：用户级偏好，存储在用户目录，个人专属（主题、快捷键等）

每个范围使用独立的设置窗口：
- `Edit/Project Settings...` → `ShowProjectSettingsWindow()` → 只显示 Project 范围的设置表
- `Edit/Preferences...` → `ShowPreferencesWindow()` → 只显示 User 范围的设置表

#### SettingsTable 基类

```csharp
/// <summary>
/// 所有设置表的基类。
/// 子类定义字段，系统自动处理序列化/UI/变更通知。
/// </summary>
public abstract class SettingsTable : INotifyPropertyChanged
{
    /// <summary>设置表唯一 ID（如 "graphics"）。</summary>
    public abstract string TableId { get; }

    /// <summary>设置表显示名称（如 "图形"）。</summary>
    public abstract string DisplayName { get; }

    /// <summary>设置范围（Project 或 User）。</summary>
    public virtual SettingsScope Scope => SettingsScope.Project;

    /// <summary>设置表图标（可选，用于 TreeView）。</summary>
    public virtual string? Icon { get; }

    /// <summary>从 JSON 字符串加载。</summary>
    public void LoadFromJson(string json) { ... }

    /// <summary>序列化为 JSON 字符串。</summary>
    public string SaveToJson() { ... }

    /// <summary>重置为默认值。</summary>
    public virtual void ResetDefaults() { ... }

    // INotifyPropertyChanged
    public event PropertyChangedEventHandler? PropertyChanged;
    protected void OnPropertyChanged([CallerMemberName] string? name = null);
}
```

```csharp
/// 不涉及文件系统，路径由调用方决定。
/// </summary>
public static class SettingsSerializer
{
    /// <summary>从 JSON 字符串反序列化。</summary>

    /// <summary>从 Stream 反序列化。</summary>

    /// <summary>序列化为 JSON 字符串。</summary>
```

```csharp
[SettingTable("graphics", "图形", Scope = SettingsScope.Project, Category = "渲染")]

{
```

## 六、SettingsModule 集成

> 注：最初设计为 SettingsFeature（IEditorFeature），但 CoreModuleImp.DiscoverFeatures 只扫描自身程序集，
> 跨程序集 Feature 不会被发现。改为静态 SettingsModule.Install()，由 EditorApplicationRunner 直接调用。

```csharp
/// <summary>设置系统模块入口——由 EditorApplicationRunner 调用 Install()。</summary>
public static class SettingsModule
{
    public static void Install()
    {
        var registry = new SettingsRegistry();
        var storage = new SettingsStorage();
        var service = new SettingsServiceImpl(registry, storage);

        // 注册内置设置表
        RegisterBuiltinSettings(service);

        // 加载已保存的设置
        service.LoadAll();

        // 注册到服务定位器
        CoreModuleImp.Context.RegisterService<ISettingsService>(service);

        // 注册菜单命令和菜单项（直接 Register，不走 Contributor）
        RegisterMenuIntegration(service);
    }

    private static void RegisterMenuIntegration(SettingsServiceImpl service)
    {
        // 注册 "editor.project_settings" 命令（Project Settings）
        var projectSettingsCmd = new EditorCommand
        {
            Id = "editor.project_settings",
            DisplayName = "Project Settings",
            Execute = _ => service.ShowProjectSettingsWindow(),
            Tooltip = "打开项目设置（图形、音频等运行时配置）"
        };
        CoreModuleImp.Context.Menus.RegisterCommand(projectSettingsCmd);

        // 注册 "editor.preferences" 命令（Preferences）
        var preferencesCmd = new EditorCommand
        {
            Id = "editor.preferences",
            DisplayName = "Preferences",
            Execute = _ => service.ShowPreferencesWindow(),
            Tooltip = "打开编辑器偏好设置（主题、快捷键等）"
        };
        CoreModuleImp.Context.Menus.RegisterCommand(preferencesCmd);

        // 注册菜单项：Edit/Project Settings...
        CoreModuleImp.Context.Menus.Register(new EditorMenuItem(
            "Edit/Project Settings...",
            CommandId: "editor.project_settings",
            SortOrder: 800,
            Icon: "⚙️"));

        // 注册菜单项：Edit/Preferences...
        CoreModuleImp.Context.Menus.Register(new EditorMenuItem(
            "Edit/Preferences...",
            CommandId: "editor.preferences",
            SortOrder: 900,
            Icon: "🔧"));
    }
}
```

```
{userHome}/.neverness/settings/{tableId}.json
例：~/.neverness/settings/editorPreferences.json

默认设置（随引擎分发）：
{engineRoot}/Engine/Config/DefaultSettings/{tableId}.json
```

注意：Project Settings 存储在项目目录，团队共享；Preferences 存储在用户目录，个人专属。

## 十二、关键技术点

### 12.1 AOT 兼容性

Phase 1-2 不用 Source Generator，但设计要预留 AOT 路径：
- SettingsSerializer 内部用 `JsonSerializer.Serialize/Deserialize` 泛型重载
- Phase 3 Generator 生成 `JsonSerializerContext` 子类
- Attribute 使用 `sealed` 类，不依赖反射创建

### 12.2 变更通知

三层通知链：
1. **SettingsTable.PropertyChanged**：字段级通知，UI 控件绑定用
2. **RuntimeSettings.SettingsChanged**：全局通知（tableId, fieldName），Runtime 模块监听用
3. **ISettingsService.SettingChanged**：Editor 服务级通知，转发自 RuntimeSettings

调用链示例：
```
用户修改 VSync
→ GraphicsSettings.VSync setter 触发 PropertyChanged("VSync")
→ RuntimeSettings.SettingsChanged("graphics", "VSync")
→ SettingsServiceImpl.RaiseSettingChanged → ISettingsService.SettingChanged
→ Renderer 监听到 → 应用新设置
```

### 12.3 重置默认值

每个 SettingsTable 实现 `ResetDefaults()` 方法，
Descriptor 提供 `CreateDefault()` 创建默认实例。
"重置默认" 按钮 → `CreateDefault()` → 复制字段值 → 触发通知。

### 12.4 TreeView 分组

按 `[SettingTable]` 的 `Category` 属性分组：
```
渲染
  ├─ 图形
  └─ 后处理
音频
  ├─ 音频设备
  └─ 音量
```
