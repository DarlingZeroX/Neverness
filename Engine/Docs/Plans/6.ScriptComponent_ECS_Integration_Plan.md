## 2. 设计决策

### 2.1 两个 TypeId 的明确区分

⚠️ **ComponentTypeId 和 ScriptTypeId 是完全不同的概念**：

| | ComponentTypeId | ScriptTypeId |
|---|---|---|
| **含义** | 组件类型的标识 | 脚本类的标识 |
| **计算** | `FNV1a64("Script")` | `FNV1a64(Type.FullName)` |
| **示例** | `0x9565553D163FC92A` | `FNV1a64("Neverness.Gameplay.PlayerController")` |
| **用途** | ECS 组件注册、ABI 调用 | 脚本实例化、序列化持久化 |
| **稳定性** | 组件名不变则不变 | FullName 不变则不变 |

**必须使用 `Type.FullName`**，不要使用 `Type.Name`。因为不同命名空间下类名可能冲突：
- `Neverness.Gameplay.PlayerController` → 不冲突
- `Neverness.UI.PlayerController` → 不冲突

**已有实现**：`ScriptRegistry.cs` 第 102 行已使用 `CalculateTypeId(type.FullName!)`。不需要新建 ScriptTypeRegistry。

### 2.2 三层结构体设计

**Native C++（ECS 存储）**：
```cpp
struct NNScriptComponent
{
    uint64_t ScriptTypeId;   // FNV1a64(Type.FullName)
    uint8_t  Enabled;        // 启用状态
    uint8_t  Reserved[7];    // 对齐填充
};  // 16 字节
```

**C# Runtime ABI（blittable 镜像）**：
```csharp
[StructLayout(LayoutKind.Sequential)]
[ComponentId(0x9565553D163FC92A, Name = "Script")]
public struct NNScriptComponentData
{
    public ulong ScriptTypeId;        // 8B
    public byte  Enabled;             // 1B（byte 对齐 C++ uint8_t）
    public fixed byte _reserved[7];   // 7B
}  // 16 字节
```

**C# Gameplay（用户 API）**：
```csharp
[StructLayout(LayoutKind.Sequential)]
public struct ScriptComponent
{
    public ulong ScriptTypeId;   // 8B
    public bool Enabled;         // 用户友好的 bool（非 ABI 字段，转换在边界层）
}  // 无需 [ComponentId]，Gameplay 组件不参与 ABI
```

### 2.3 关键规则

| 规则 | 说明 |
|------|------|
| ComponentTypeId ≠ ScriptTypeId | `FNV1a64("Script")` vs `FNV1a64(Type.FullName)` |
| ScriptTypeId 用 FullName | `Neverness.Gameplay.PlayerController`，不用短名 |
| Enabled 保持 uint8_t | 不为 Reflection 妥协 ECS 数据模型 |
| Reserved 不注册反射 | ABI padding，Inspector 不显示 |
| BehaviourRegistry 不改 | 保持 `Dictionary<ulong, List<EntityBehaviour>>` |
| Gameplay 组件无 [ComponentId] | 与其他 Gameplay 组件保持风格统一 |
| Gameplay 用 bool Enabled | 用户 API 友好，转换在 ABI 边界层 |
| 一 Entity 一 Script | 当前限制，未来扩展暂不设计 |

### 2.4 职责边界

```
Native ECS → 序列化、Prefab、场景保存、反射、Inspector
C# Scheduler → 实例创建、OnCreate、OnUpdate、OnDestroy、HotReload
```

### 2.5 反射注册

```cpp
NN_FIELD(NNScriptComponent, ScriptTypeId, UInt64)
NN_FIELD(NNScriptComponent, Enabled, UInt32)  // TODO: Reflection 增加 Bool/UInt8 后改为真实类型
```

**不注册 Reserved**。

## 6. 风险与注意事项

1. ScriptTypeId = `FNV1a64(Type.FullName)`，不用 Type.Name
2. Enabled 保持 uint8_t，Reflection 注册为 UInt32 是临时方案，加 TODO 标记
3. Reserved 不注册反射
4. BehaviourRegistry 保持 List 映射不动
5. Gameplay 组件无 [ComponentId]，与其他 Gameplay 组件风格统一
6. Gameplay 用 bool Enabled，转换在 ABI 边界层
7. 新增组件不影响 VGSC 版本号（反射驱动序列化自动处理）
