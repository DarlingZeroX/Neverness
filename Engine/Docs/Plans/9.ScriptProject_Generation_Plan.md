# 脚本工程文件生成（.sln + .csproj）— 实施计划 v3

> **日期**: 2026-06-02
> **状态**: ✅ 实施完成（运行时验证待做）
> **模块**: Neverness.Editor.Script + Neverness.Gameplay

---

## 1. 目标

像 Unity 一样，在项目根目录生成持久化的 `.sln` 和 `.csproj` 工程文件：
- 用户可以在 IDE（VS / Rider）中打开 `.sln` 直接编辑和调试脚本
- 编译使用持久化的 `.csproj` 而不是临时生成的
- 增量编译支持（dotnet build 内置）
- 工程文件生成与编译解耦

---

## 2. 关键设计决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 解决方案文件 | `Game.sln`（不叫 Assembly-CSharp.sln） | Solution 代表项目，不代表程序集 |
| 源文件引用 | `<Compile Include="Assets\**\*.cs" />` glob | 增删文件不需要重新生成 csproj |
| 工程生成与编译解耦 | Editor 启动/脚本增删 → Generate，编译 → 直接 build | 单一职责 |
| 引擎引用 | `ScriptAssemblyDefinition.References` 描述依赖名，Resolver 只负责名字→路径 | 不耦合具体程序集列表 |
| .sln GUID | 确定性生成（基于 assembly name 的 MD5） | 避免 VS 每次重新加载 |
| 编译 API | `Compile(ScriptAssemblyDefinition)` | 统一 API，为未来多 assembly 预留 |
| TargetFramework | 跟随宿主运行时（从 EntityBehaviour.Assembly 读取） | 避免升级时忘记同步 |
| GenerateSolution | 接受 `IReadOnlyList<ScriptAssemblyDefinition>` | 为未来多 csproj 预留 |

## 4. 引擎程序集引用

### 4.1 KnownEngineAssemblies

不扫描 AppDomain，使用静态列表：

```csharp
/// <summary>已知的引擎程序集名称列表。</summary>
private static readonly string[] KnownEngineAssemblies =
[
    "Neverness.Gameplay",
    "Neverness.Runtime.Scene",
    "Neverness.Runtime.Engine",
    "Neverness.Runtime.Assets",
    "Neverness.Runtime.Interop",
];
```

### 4.2 IEngineAssemblyResolver

```csharp
/// <summary>
/// 引擎程序集解析器——根据已知名称查找 DLL 路径。
/// 使用相对路径 HintPath，不写死绝对路径。
/// </summary>
public interface IEngineAssemblyResolver
{
    IReadOnlyList<EngineAssemblyReference> Resolve();
}

public sealed class EngineAssemblyReference
{
    public required string Name { get; init; }
    /// <summary>DLL 相对于项目根目录的路径（如 Engine/Binaries/Neverness.Gameplay.dll）。</summary>
    public required string RelativeDllPath { get; init; }
}
```

### 4.3 DefaultEngineAssemblyResolver

```csharp
public sealed class DefaultEngineAssemblyResolver : IEngineAssemblyResolver
{
    private readonly string _engineBinariesDir; // 引擎 DLL 目录（OS 绝对路径）

    public IReadOnlyList<EngineAssemblyReference> Resolve()
    {
        var result = new List<EngineAssemblyReference>();
        foreach (var name in KnownEngineAssemblies)
        {
            var dllPath = Path.Combine(_engineBinariesDir, $"{name}.dll");
            if (File.Exists(dllPath))
            {
                result.Add(new EngineAssemblyReference
                {
                    Name = name,
                    RelativeDllPath = GetRelativePath(dllPath)
                });
            }
        }
        return result;
    }
}
```

## 5. .csproj 格式

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net10.0</TargetFramework>
    <AssemblyName>Assembly-CSharp</AssemblyName>
    <OutputType>Library</OutputType>
    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
    <EnableDefaultCompileItems>false</EnableDefaultCompileItems>
  </PropertyGroup>

  <!-- 引擎程序集引用（相对路径） -->
  <ItemGroup>
    <Reference Include="Neverness.Gameplay">
      <HintPath>Engine\Binaries\Neverness.Gameplay.dll</HintPath>
      <Private>false</Private>
    </Reference>
    <!-- ... -->
  </ItemGroup>

  <!-- 源文件 glob -->
  <ItemGroup>
    <Compile Include="Assets\**\*.cs" />
  </ItemGroup>
</Project>
```

## 6. .sln 格式（确定性 GUID）

```csharp
private string GenerateSlnContent(string assemblyName, string csprojFileName)
{
    // 确定性 GUID：基于 assembly name 的 MD5
    var projectGuid = GenerateDeterministicGuid(assemblyName).ToString("B").ToUpperInvariant();

    return $"""
        Microsoft Visual Studio Solution File, Format Version 12.00
        # Visual Studio Version 17
        VisualStudioVersion = 17.0.31903.59
        MinimumVisualStudioVersion = 10.0.40219.1
        Project("{{FAE04EC0-301F-11D3-BF4B-00C04F79EFBC}}") = "{assemblyName}", "{csprojFileName}", "{projectGuid}"
        EndProject
        Global
        	GlobalSection(SolutionConfigurationPlatforms) = preSolution
        		Debug|Any CPU = Debug|Any CPU
        		Release|Any CPU = Release|Any CPU
        	EndGlobalSection
        	GlobalSection(ProjectConfigurationPlatforms) = postSolution
        		{projectGuid}.Debug|Any CPU.ActiveCfg = Debug|Any CPU
        		{projectGuid}.Debug|Any CPU.Build.0 = Debug|Any CPU
        		{projectGuid}.Release|Any CPU.ActiveCfg = Release|Any CPU
        		{projectGuid}.Release|Any CPU.Build.0 = Release|Any CPU
        	EndGlobalSection
        EndGlobal
        """;
}

/// <summary>基于名称生成确定性 GUID（MD5）。</summary>
private static Guid GenerateDeterministicGuid(string name)
{
    using var md5 = System.Security.Cryptography.MD5.Create();
    var hash = md5.ComputeHash(System.Text.Encoding.UTF8.GetBytes($"Neverness.ScriptProject.{name}"));
    return new Guid(hash);
}
```

## 7. ScriptAssemblyDefinition

```csharp
/// <summary>
/// 脚本程序集定义——描述一个可编译的脚本程序集。
/// 当前只有 Assembly-CSharp，未来支持 asmdef 多程序集。
/// </summary>
public sealed class ScriptAssemblyDefinition
{
    /// <summary>程序集名称（如 Assembly-CSharp）。</summary>
    public required string Name { get; init; }

    /// <summary>源文件 glob 模式列表（如 ["Assets/**/*.cs"]）。</summary>
    public required IReadOnlyList<string> SourceGlobs { get; init; }

    /// <summary>引用的程序集名称列表（如 ["Neverness.Gameplay"]）。</summary>
    public required IReadOnlyList<string> References { get; init; }

    /// <summary>输出目录。</summary>
    public required string OutputPath { get; init; }
}
```
