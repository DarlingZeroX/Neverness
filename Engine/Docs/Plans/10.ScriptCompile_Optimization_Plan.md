# 脚本编译优化 — 减少 Play Mode 等待时间

> **日期**: 2026-06-02
> **状态**: ✅ 批准实施

---

## 1. 核心思路

```
始终执行 dotnet build（让 MSBuild 决定是否需要编译）
  → DLL 没变 → 跳过 ALC 加载，复用已加载 Assembly
  → DLL 变了 → 重新加载 Assembly
```

**不和 MSBuild 竞争**。源文件变更检测交给 MSBuild 增量编译，自己只做 DLL 缓存。

---

## 2. 当前问题

每次 EnterPlayMode 都执行：
1. `dotnet build`（2-5s，即使无变更也要启动进程 ~300-1000ms）
2. `ScriptAssemblyLoader.Load()`（~0.5s，即使 DLL 没变）
3. `RegisterTypesFromAssembly()`（~0.1s，即使类型没变）

---

## 3. 优化：只做 DLL 缓存 + Assembly 复用

```
EnterPlayMode
  ↓
dotnet build（始终执行，MSBuild 增量判断）
  ↓
DLL 变化？
  ├─ 否 + Assembly 已缓存 → 直接进入 Play（跳过 Load + Register）
  └─ 是 → Load Assembly → Register Types → 缓存
```

### 预期效果

| 场景 | 当前 | 优化后 |
|------|------|--------|
| 无变更再次 Play | 2-5s | **0.3-1s**（dotnet build 增量检查 + 跳过 Load） |
| 修改 1 个 .cs | 2-5s | **1-2s**（dotnet build 增量编译 + Reload） |
| 首次 Play | 2-5s | 2-5s（必须编译） |

---

## 4. 修改

只改 `ScriptCompileService.cs`，其他文件不变。

```csharp
public sealed class ScriptCompileService
{
    private readonly ScriptCompiler _compiler = new();
    private readonly ScriptAssemblyLoader _assemblyLoader = new();

    // ── Assembly 缓存 ──
    private string? _loadedDllPath;
    private DateTime _loadedDllTimestamp;
    private Assembly? _cachedAssembly;

    public Public.ScriptCompileResult Compile(ScriptAssemblyDefinition assembly)
    {
        var stopwatch = System.Diagnostics.Stopwatch.StartNew();

        var projectRoot = VFS.GetAbsolutePath("/");
        if (string.IsNullOrEmpty(projectRoot))
        {
            return ErrorResult("VFS project root not resolvable.", stopwatch.Elapsed);
        }

        var csprojPath = Path.Combine(projectRoot, $"{assembly.Name}.csproj");
        var outputPath = Path.Combine(
            VFS.GetAbsolutePath(ProjectPaths.Library.FullPath),
            "Scripts");
        var dllPath = Path.Combine(outputPath, $"{assembly.Name}.dll");

        // 1. 始终执行 dotnet build（MSBuild 自己判断是否需要编译）
        var result = _compiler.Compile(csprojPath, outputPath, assembly.Name, assembly.Configuration);

        if (!result.Success)
        {
            var errorResult = ToEditorResult(result, stopwatch.Elapsed);
            CompileFinished?.Invoke(this, errorResult);
            return errorResult;
        }

        // 2. 检查 DLL 是否变化
        var currentDllTimestamp = File.Exists(dllPath)
            ? File.GetLastWriteTimeUtc(dllPath)
            : DateTime.MinValue;

        if (_cachedAssembly != null &&
            string.Equals(_loadedDllPath, dllPath, StringComparison.OrdinalIgnoreCase) &&
            currentDllTimestamp == _loadedDllTimestamp)
        {
            // DLL 未变化 → 跳过重新加载
            stopwatch.Stop();
            Console.WriteLine($"[ScriptCompileService] DLL unchanged, skip reload ({stopwatch.ElapsedMilliseconds}ms)");
            var cachedResult = new Public.ScriptCompileResult
            {
                Status = Public.ScriptCompileStatus.Success,
                Diagnostics = Array.Empty<Public.ScriptDiagnostic>(),
                Duration = stopwatch.Elapsed
            };
            CompileFinished?.Invoke(this, cachedResult);
            return cachedResult;
        }

        // 3. DLL 变化 → 重新加载
        _assemblyLoader.Load(dllPath);
        RegisterTypesFromAssembly(_assemblyLoader.LoadedAssembly!);

        // 4. 更新缓存
        _loadedDllPath = dllPath;
        _loadedDllTimestamp = currentDllTimestamp;
        _cachedAssembly = _assemblyLoader.LoadedAssembly;

        stopwatch.Stop();
        Console.WriteLine($"[ScriptCompileService] Assembly reloaded ({stopwatch.ElapsedMilliseconds}ms)");
        var successResult = ToEditorResult(result, stopwatch.Elapsed);
        CompileFinished?.Invoke(this, successResult);
        return successResult;
    }

    // HotReload 保持不变（强制重新加载）
    public Public.ScriptCompileResult HotReload(ScriptAssemblyDefinition assembly)
    {
        var scheduler = GameplayContext.Current?.BehaviourScheduler;
        var behaviourRegistry = scheduler?.Registry;

        if (scheduler != null && behaviourRegistry != null)
        {
            foreach (var behaviour in behaviourRegistry.AllBehaviours.ToList())
                scheduler.MarkForDestroy(behaviour);
            behaviourRegistry.ProcessPendingDestroy();
        }

        // 清除缓存，强制重新加载
        _cachedAssembly = null;
        _loadedDllPath = null;

        return Compile(assembly);
    }

    // ... 其他方法不变
}
```

---

## 5. 关键点

1. **始终调用 dotnet build** — 不做源文件变更检测，让 MSBuild 增量编译处理
2. **DLL 变化检测** — 用 `File.GetLastWriteTimeUtc(dllPath)` 比较时间戳
3. **Assembly 复用** — DLL 没变时跳过 `Load()` + `RegisterTypesFromAssembly()`
4. **HotReload 清除缓存** — 强制重新编译 + 加载
5. **不持久化缓存** — 内存缓存，Editor 重启后首次 Play 会正常编译
