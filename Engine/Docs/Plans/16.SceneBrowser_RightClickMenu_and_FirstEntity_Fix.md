## 修复 1：右键菜单——空白区域误显示 Entity 菜单

### 根因

`OnTreeViewContextRequested` 使用 `_treeView.SelectedItem` 判断右键位置。但 Avalonia 的 `TreeView.SelectedItem` 在右键点击空白区域时**不会清除**，保留着上次左键选中的节点。导致空白区域右键永远走进 Entity 菌单分支。

### 修复

改用 `e.Source`（事件实际源头控件）向上查找 `TreeViewItem`：

```csharp
// 旧代码（错误）
if (_treeView.SelectedItem is TreeViewItem treeItem && treeItem.Tag is int handle && handle > 0)

// 新代码（正确）
var hitItem = FindParentTreeViewItem(e.Source as Control);
if (hitItem != null && hitItem.Tag is int handle && handle > 0)

// 辅助方法
private static TreeViewItem? FindParentTreeViewItem(Control? control)
{
    while (control != null)
    {
        if (control is TreeViewItem tvi) return tvi;
        control = control.Parent as Control;
    }
    return null;
}
```

## 修复 2：首次添加 Entity 不显示（延迟一帧）

### 根因

**事件发射时机太早**——在组件挂载之前就发射了 `EntityCreated` 事件。

调用链（修复前）：

```
EntityFactory.CreateCamera(world)
  → world.CreateEntity("Camera")
    → FrifloScene.CreateEntity()     ← 创建实体（仅 TagComponent + RelationshipComponent）
    → Events.Emit(EntityCreated)     ← 事件触发！此时没有 TransformComponent！
  → entity.Add(TransformComponent)   ← 事件已过，缓存不会更新
  → entity.Add(CameraComponent)
```

当 `EntityCreated` 事件触发时，`SceneHierarchyCache.RebuildFromScene` 查询 `Query<TransformComponent>()`——新实体没有 `TransformComponent`，查不到。

用户行为对应：
- 添加 Entity 1 → 事件触发 → 缓存查询找不到 Entity 1（无 TransformComponent）→ 不显示
- 添加 Entity 2 → 事件触发 → 缓存查询找到 Entity 1（已有 TransformComponent）→ Entity 1 显示，Entity 2 不显示
- 依此类推，永远延迟一帧

### 修复

将事件发射移至组件挂载**之后**。

#### SceneWorld.cs

```csharp
// 旧代码
public SceneEntity? CreateEntity(string? displayName = null)
{
    var entity = _scene.CreateEntity(displayName);
    Events.Emit(SceneEvent.OnEntityCreated(entity));  // ← 太早！
    return new SceneEntity(entity, _scene);
}

// 新代码
public SceneEntity? CreateEntity(string? displayName = null)
{
    var entity = _scene.CreateEntity(displayName);
    // 不发射事件，由调用方在组件挂载后手动调用 EmitEntityCreated
    return new SceneEntity(entity, _scene);
}

// 新增方法
public void EmitEntityCreated(IEntity entity)
{
    Events.Emit(SceneEvent.OnEntityCreated(entity));
}
```

#### EntityFactory.cs

所有 `CreateXxx` 方法在末尾添加：

```csharp
// 组件挂载完毕，发射事件（此时缓存查询能找到带 TransformComponent 的实体）
if (scene is SceneWorld world)
    world.EmitEntityCreated(entity);
```

#### SceneQueryServiceImpl.cs

`CreateChildEntity` 和 `DuplicateEntity` 需要：
1. 添加 `TransformComponent`
2. 手动发射事件

```csharp
var entity = _activeScene.CreateEntity();

// 添加 TransformComponent（SceneHierarchyCache 查询依赖此组件）
entity.Add(new TransformComponent
{
    Position = System.Numerics.Vector3.Zero,
    Rotation = System.Numerics.Quaternion.Identity,
    Scale = System.Numerics.Vector3.One,
});

// 组件挂载完毕，发射事件
if (_activeScene is SceneWorld world)
    world.EmitEntityCreated(entity);
```

### 执行顺序（修复后）

```
EntityFactory.CreateCamera(world)
  → world.CreateEntity("Camera")        ← 创建实体，不发射事件
  → entity.Add(TransformComponent)       ← 挂载组件
  → entity.Add(CameraComponent)
  → world.EmitEntityCreated(entity)      ← 组件齐全后发射事件
    → OnSceneEvent → RefreshTree
      → RebuildFromScene → Query<TransformComponent>
        ← 找到新实体（有 TransformComponent）✓
      → ViewModel 更新 → TreeView 刷新 → 实体立即可见 ✓
```

## 附带修复：Avalonia UI 线程安全

### 问题

事件从主线程触发（`SceneWorld.CreateEntity → Emit → RefreshTree → OnPropertyChanged`），但 Avalonia 控件只能在 UI 线程修改。

### 修复

`OnPropertyChanged` 中用 `Dispatcher.UIThread.Invoke` 确保在 UI 线程执行：

```csharp
private void OnPropertyChanged(string propertyName)
{
    if (propertyName == nameof(SceneBrowserViewModel.VisibleNodes))
    {
        if (Dispatcher.UIThread.CheckAccess())
            RebuildTreeView();
        else
            Dispatcher.UIThread.Invoke(RebuildTreeView);
    }
}
```
