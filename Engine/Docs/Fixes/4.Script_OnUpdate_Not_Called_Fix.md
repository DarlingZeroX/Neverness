# 4. 脚本 OnUpdate 不被调用修复

**日期**: 2026-06-22
**问题**: 进入 Play Mode 后，EntityBehaviour 的 OnUpdate 方法不被调用，Debug.Log 无输出
**影响**: 所有用户脚本无法执行

---

## 问题现象

用户在场景中挂载了 NewScript.cs（继承 EntityBehaviour），进入 Play Mode 后控制台没有 `[INFO] NewScript: OnUpdate1` 输出。

---

## 根本原因

经过逐步调试，发现 **3 个独立问题** 共同导致了此 bug：

### 问题 1: `Tags` 属性缺失

**位置**: `ScriptBehaviourBridge.cs`, `ScriptBehaviourScheduler.cs`

**原因**: 两个类都标记了 `[SceneSystemTag(SceneSystemTags.Gameplay)]`，但没有实现 `ISceneSystem.Tags` 属性。接口默认值是 `SceneSystemTags.All`。

**影响**: `FrifloScene.UpdateByTagMask()` 检查的是 `system.Tags` 属性，不是 Attribute 标记。由于 `Tags` 默认是 `All`，系统在所有模式下都会执行，但 Editor 侧的 Play Mode 逻辑没有正确触发。

**修复**: 在两个类中添加 `Tags` 属性实现：
```csharp
public SceneSystemTags Tags => SceneSystemTags.Gameplay;
```

---

### 问题 2: `behaviour.Enabled` 被错误设为 `false`

**位置**: `ScriptBehaviourBridge.cs` 第 145 行

**原因**: 创建 Behaviour 时执行了：
```csharp
behaviour.Enabled = script.IsInitialized;
```

`ScriptComponent.IsInitialized` 标记了 `[Transient]` 属性，序列化/反序列化时被跳过，永远是 `false`。

**影响**: `behaviour.Enabled = false`，导致 `ScriptBehaviourScheduler.OnUpdate()` 中的检查失败：
```csharp
if (behaviour.Enabled && !behaviour.IsDestroyed && behaviour.IsStarted)
{
    behaviour.OnUpdate(deltaTime);  // 永远不会执行
}
```

**修复**: 创建 Behaviour 时直接设为 `true`：
```csharp
behaviour.Enabled = true;
```

---

### 问题 3: `IScene` 到 `SceneWorld` 类型转换失败

**位置**: `ScriptBehaviourBridge.Update()` 第 85 行

**原因**: 原代码：
```csharp
public void Update(float deltaTime)
{
    if (_scene is SceneWorld world)
    {
        Sync(world);
    }
}
```

FrifloScene 调用 `system.Initialize(this)` 时传入的是 `FrifloScene`（实现 `IScene`），不是 `SceneWorld`。所以 `_scene is SceneWorld` 返回 `false`，`Sync()` 永远不会被调用。

**影响**: 整个脚本同步逻辑被跳过，Behaviour 不会被创建。

**修复**: 通过回调函数获取 SceneWorld：
```csharp
private readonly Func<SceneWorld?> _getWorld;

public ScriptBehaviourBridge(
    IScriptFactory factory,
    BehaviourRegistry registry,
    ScriptRegistry scriptRegistry,
    ScriptBehaviourScheduler scheduler,
    Func<SceneWorld?> getWorld)
{
    _factory = factory;
    _registry = registry;
    _scriptRegistry = scriptRegistry;
    _scheduler = scheduler;
    _getWorld = getWorld;
}

public void Update(float deltaTime)
{
    var world = _getWorld();
    if (world != null && world.IsValid)
    {
        Sync(world);
    }
}
```

## 经验教训

1. **Attribute ≠ 属性**: `[SceneSystemTag]` 只是标记，不会自动设置 `Tags` 属性，需要手动实现
2. **[Transient] 字段不可用于控制逻辑**: 序列化后永远是默认值，不能用于运行时状态控制
3. **接口继承的隐式转换**: `IScene` 到具体类型 `SceneWorld` 的转换需要显式处理，不能依赖隐式转换
4. **逐步调试**: 添加日志时应从调用链顶端开始，逐层定位断点
