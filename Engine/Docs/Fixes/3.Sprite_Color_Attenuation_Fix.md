# 3. Sprite 颜色衰减修复

**日期：** 2026-06-20
**症状：** Viewport 中 Sprite 渲染的颜色比图片查看器暗 7-10 倍（如 sRGB 37→5），纯白正常。
**根因：** FBO 使用 `RGBA8_UNORM` 格式，纹理采样后的线性值直接存为字节，CopyTexture 到 SRGB SwapChain 时被当 sRGB 解读，暗色被双重线性化。

---

## 问题分析

### 颜色空间链路（修复前）

```
纹理(SRGB) → GPU采样 auto sRGB→linear → Shader(linear) → UNORM FBO(存线性字节) → CopyTexture → SwapChain(SRGB)
```

SwapChain 是 SRGB 格式，Present 时监视器把字节当 sRGB 解读。线性值 0.017 存为字节 ~4，显示为 sRGB 4，而正确值应该是 37。

### 衰减数学验证

```
sRGB 37 → 线性化 (37/255)^2.2 ≈ 0.017 → 存为字节 ~4 → 显示 4（÷9.25）
白色 255 → 线性 1.0 → 存为字节 255 → 显示 255（不变）
```

完美匹配观察到的 ÷7~10 衰减。

---

## 修复方案

### 方案 A（第一次尝试，失败）

把 FBO 改为 `RGBA8_SRGB`，让 GPU 写入时自动 linear→sRGB。

**结果：** Sprite 颜色正确，但 RmlUI overlay 颜色变亮 + PSO 格式不匹配警告。

**失败原因：** RmlUI 整个管线故意在 sRGB 空间工作（GL3 后端明确 `glDisable(GL_FRAMEBUFFER_SRGB)`），FBO 改为 SRGB 后 RmlUI 输出的 sRGB 值被 GPU 再做一次 linear→sRGB，导致双重编码。

### 方案 B（最终方案，成功）

FBO 保持 `RGBA8_UNORM`（兼容 RmlUI），在 Sprite 像素着色器输出前手动做 linear→sRGB 转换。

**颜色空间链路（修复后）：**

```
纹理(SRGB) → GPU采样 auto sRGB→linear → Shader(linear) → LinearToSRGB → UNORM FBO(存sRGB字节) → CopyTexture → SwapChain(SRGB) ✓
RmlUI(sRGB空间) → 直接输出 → UNORM FBO(存sRGB字节) → 同上 ✓
```

两个管线共享同一个 UNORM FBO，RmlUI 不受影响。

---

## 修改文件

### 1. Sprite 像素着色器（核心修复）

**文件：** `Engine/Source/Runtime/NNRuntimeRenderer2D/Include/Renderer2D/BuiltinShaders.h`

```hlsl
// 新增：线性 → sRGB 转换函数
float3 LinearToSRGB(float3 lin)
{
    return float3(
        lin.r <= 0.0031308 ? lin.r * 12.92 : 1.055 * pow(lin.r, 1.0 / 2.4) - 0.055,
        lin.g <= 0.0031308 ? lin.g * 12.92 : 1.055 * pow(lin.g, 1.0 / 2.4) - 0.055,
        lin.b <= 0.0031308 ? lin.b * 12.92 : 1.055 * pow(lin.b, 1.0 / 2.4) - 0.055);
}

// 修改：输出前做 linear→sRGB
float4 main(PSInput input) : SV_Target
{
    float4 texColor = u_Texture.Sample(u_Texture_sampler, input.TexCoord);
    float4 result = texColor * u_Color;
    return float4(LinearToSRGB(result.rgb), result.a);
}
```

### 2. 无需修改的文件

- `FramebufferObject.cpp` — FBO 保持 `RGBA8_UNORM`
- `Renderer2D.cpp` — RenderPass/ClearValue 保持 `TEX_FORMAT_RGBA8_UNORM`
- `RmlUIRenderer.cpp` — RmlUI FBO 保持 `RGBA8_UNORM`

## 经验教训

1. **不能只看单个管线。** Sprite 和 RmlUI 共享同一个 FBO，改 FBO 格式会影响所有管线。
2. **sRGB 格式是"自动转换"，不是"标记"。** `RGBA8_SRGB` 意味着 GPU 读时自动 sRGB→linear，写时自动 linear→sRGB。如果 shader 输出已经是 sRGB 空间，写入 SRGB FBO 会双重编码。
3. **RmlUI 故意在 sRGB 空间工作。** 这是 web/CSS 的惯例，不能随便改成线性空间。
4. **HLSL 保留字。** `linear` 是 HLSL 关键字，不能用作变量名。
5. **精确 sRGB 曲线 vs pow(2.2)。** 使用 `1.055 * pow(x, 1/2.4) - 0.055` 比简单 `pow(x, 1/2.2)` 更精确，尤其在低值区域。
