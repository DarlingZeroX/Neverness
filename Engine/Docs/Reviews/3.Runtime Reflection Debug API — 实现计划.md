Runtime Reflection Debug API — 实现计划

 Context

 当前 Editor 已有 hierarchy snapshot + transform snapshot，但 C# Inspector 无法通过 Runtime
 Reflection 自动构建 UI。
 需要新增「结构化 Reflection Snapshot API」——Native 只输出 POD reflection metadata + raw
 component data，
 C# 通过这些数据自动构建 Inspector / Property Grid / Component Viewer。

 核心原则：零硬编码组件类型，所有数据来自 NNComponentRegistry。

 /**
  * @brief 组件类型信息——描述单个组件类型的元数据。24 字节。
  * nameOffset + nameLen 引用 snapshot 内嵌的 namePool。
  */
 typedef struct NNEditorComponentInfo
 {
     uint64_t typeId;        ///< FNV-1a name hash，与 NNSceneAPI 的 componentTypeId 一致
     uint32_t nameOffset;    ///< 在 namePool 中的字节偏移
     uint32_t nameLen;       ///< 名字字节长度（不含 NUL）
     uint32_t fieldCount;    ///< 该组件的字段数量
     uint32_t flags;         ///< 保留（未来：[0]=serializable [1]=hasPostDeserialize ...）
 } NNEditorComponentInfo;

 NN_STATIC_ASSERT(sizeof(NNEditorComponentInfo) == 24, "...");

 /**
  * @brief 组件字段信息——描述单个字段的反射元数据。24 字节。
  * nameOffset + nameLen 引用 snapshot 内嵌的 namePool。
  */
 typedef struct NNEditorFieldInfo
 {
     uint32_t nameOffset;    ///< 在 namePool 中的字节偏移
     uint32_t nameLen;       ///< 名字字节长度（不含 NUL）
     uint32_t fieldType;     ///< NNComponentFieldType 枚举值（cast to uint32_t）
     uint32_t dataOffset;    ///< 字段在组件原始数据中的字节偏移

     uint32_t _pad;          ///< 对齐到 8 字节边界

 Phase 2: NNReflectionSnapshotBuilder — 类型信息快照构建器

 新建文件: Engine/Source/Runtime/NNRuntimeScene/Include/Snapshot/NNReflectionSnapshotBuilder.h
 新建文件:
 Engine/Source/Runtime/NNRuntimeScene/Source/Snapshot/NNReflectionSnapshotBuilder.cpp

 跟随 NNHierarchySnapshotBuilder 的 EstimateSize + Build 模式。

 2.1 内存布局

 [ NNSceneSnapshotHeader        32 bytes ]
 [ NNEditorComponentInfo[]      24 bytes × typeCount ]
 [ NNEditorFieldInfo[]          24 bytes × totalFieldCount ]
 [ char namePool[]              namePoolBytes, UTF-8 NUL-separated ]

 - header.magic = 0x56475343（复用 VGSC magic）
 - header.layoutVersion = 2（类型信息快照格式版本，区别于 hierarchy 的 1）
 - header.nodeCount = typeCount（组件类型数量）
 - header.namePoolBytes = 名字池总字节数
 - header.hierarchyVersion = scene.ReflectionVersion()
 - 其余 header 字段保留为 0

 2.2 EstimateSize

 遍历 registry.ForEachDescriptor:
     if desc.Fields.empty() → skip
     typeCount++
     totalFieldCount += desc.Fields.size()
     namePoolBytes += strlen(desc.NameUtf8) + 1
     for field in desc.Fields:
         namePoolBytes += strlen(field.NameUtf8) + 1

 return sizeof(Header) + typeCount * 24 + totalFieldCount * 24 + namePoolBytes

 2.3 Build（两遍算法）

 Pass 1 = EstimateSize（同上，收集 typeCount, totalFieldCount, namePoolBytes）
 capacity 检查：不足则返回所需大小

 Pass 2 — 写入：
     nameOffset = 0
     for each descriptor (有字段的):
         写入 NNEditorComponentInfo:
             typeId = desc.NameHash
             nameOffset = 当前偏移
             nameLen = strlen(desc.NameUtf8)
             fieldCount = desc.Fields.size()
             flags = 0
         namePool 写入 desc.NameUtf8 + '\0'
         nameOffset += nameLen + 1

     for each descriptor (有字段的):
         for each field in desc.Fields:
             写入 NNEditorFieldInfo:
                 nameOffset, nameLen
                 fieldType = (uint32_t)field.FieldType
                 dataOffset = field.Offset
                 dataSize = field.Size
                 _pad = 0
             namePool 写入 field.NameUtf8 + '\0'

 实现要点：

 - GetReflectionVersion：返回 s->ReflectionVersion()
 - GetTypeInfoSnapshotSize / GetTypeInfoSnapshot：委托 NNReflectionSnapshotBuilder
 - GetEntityComponentCount：遍历 registry.ForEachDescriptor，对每个有 HasComponentFn
 的描述符检查实体是否有该组件
 - GetEntityComponents：同上，写入 NNEditorComponentInfo 数组（只有 typeId 和 flags 有效，name
  通过类型快照获取）
 - GetComponentFieldInfos：registry.FindDescByNameHash(typeId) 遍历 desc.Fields，写入
 NNEditorFieldInfo 数组
 - GetComponentRawData：通过 desc.GetComponentConstPtrFn 获取组件指针，memcpy 到调用方 buffer
