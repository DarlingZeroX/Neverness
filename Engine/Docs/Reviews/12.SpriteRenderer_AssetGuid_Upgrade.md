# SpriteRenderer AssetId: uint64_t → NNGuid (128-bit) 升级

## 目标

将 `NNSpriteRendererComponent` 的 `TextureAsset` / `MaterialAsset` 从 `uint64_t`（仅 GUID low）升级为完整 `NNGuid`（128-bit），修复序列化 bug、预加载盲区和 Inspector 数据丢失问题。

## 问题分析

1. **二进制序列化 bug**：`NNSceneSerializer.cpp` 中 `UInt64` 和 `Entity` 共用 case，asset ID 被错误地当 entity 引用做 `EntityToArchiveIndex` 重映射
2. **预加载盲区**：`NNSceneAssetLoader` 只扫描 `Guid` 类型字段，`UInt64` 类型的 asset 引用不可见
3. **Inspector 数据丢失**：拖拽纹理时 `droppedGuid.Low` 丢掉了 high 半部分

## 设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| C# 结构体 | 用同文件 `NNGuid` struct | blittable，与 C++ `NNGuid` 布局一致，不反向依赖 Assets 程序集 |
| RenderAssetManager 缓存键 | 内部仍用 `guid.low` | 缓存是 runtime-only，用 real UUID 时碰撞概率极低 |
| `LoadAssetByGuidLow` | 保留，SpriteRenderSystem 改用 `LoadAssetSync(NNGuid)` | 其他代码可能仍在用 |
| 格式版本 | 二进制 2→3，JSON 1→2 | wire format 不兼容，必须升版本 |
| 旧场景迁移 | v2 降级读取（LEGA 标记） | 旧 v2 文件因 bug 已损坏，标记后让用户重新指定 |
