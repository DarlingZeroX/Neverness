# Play Mode 实现文档

> Neverness 第一代 Play Mode 架构——实施完成记录。

## 概述

Play Mode 允许编辑器进入播放模式运行场景，退出时自动恢复到播放前的状态。核心设计原则：**Runtime 不知道 PlayMode 的存在**，所有播放模式逻辑由 Editor 侧驱动。

## 核心设计

### 1. SceneSystemTags（Runtime 侧）

**文件**: `Runtime.Scene/Public/Systems/SceneSystemTags.cs`

只定义原子标签，不定义组合标签：

```csharp
[Flags]
public enum SceneSystemTags : ushort
{
    None      = 0,
    Editor    = 1 << 0,
    Gameplay  = 1 << 1,
    Render    = 1 << 2,
    Audio     = 1 << 3,
    Physics   = 1 << 4,
    Streaming = 1 << 5,
}
```

默认值通过 `SceneSystemTagDefaults.Default` 常量定义：`Render | Audio | Streaming`。

系统通过 `[SceneSystemTag(SceneSystemTags.Gameplay)]` 特性标记标签，未标记的系统默认 `Always`。

### 2. PlayMode 状态机（Editor 侧）

**文件**: `Editor.Scene/Private/PlayMode/PlayModeController.cs`

```
Editing ──EnterPlay──► Playing ──Pause──► Paused
   ▲                     │                  │
   └─────ExitPlay────────┘◄──Resume─────────┘
```

每个模式对应不同的 tagMask：

| 模式 | tagMask |
|------|---------|
| Editing | `Editor \| Render \| Audio \| Streaming` |
| Playing | `Gameplay \| Physics \| Render \| Audio \| Streaming` |
| Paused | `Render \| Audio \| Streaming` |

### 3. 快照机制

进入播放前，将当前场景序列化到临时文件：

```
Cache/PlayModeSnapshots/{guid}.vgsc
```

使用 `Guid.NewGuid()` 命名，避免冲突。退出播放时通过 `SceneManager.ReplaceActiveSceneFromSnapshot()` 原子恢复，不触发事件风暴。

### 4. Tick 驱动链

```
RuntimeMainLoop.Tick(deltaTime)
  → RuntimeLoop.Tick
    → SceneSubsystem.Tick
      → PlayModeController.TickActiveScene(deltaTime)  // 通过 TickOverride 委托
        → SceneWorld.TickByTagMask(deltaTime, mask)
          → Scheduler.TickByTagMask(group, mask, deltaTime)  // Managed 按标签过滤
          → NativeBridge.TickSystems(handle, deltaTime, mask) // Native 按标签过滤
          → Events.FlushDeferred(mask)                        // 按标签 Flush
```

### 5. 命令系统解耦

Shell 模块（Level 0）不直接引用 Scene 模块（Level 2）。通过 `EditorMenuRegistry` 命令系统解耦：

| 命令 ID | 说明 |
|---------|------|
| `scene.play` | 进入播放 / 恢复播放 |
| `scene.stop` | 退出播放 |
| `scene.pause` | 暂停 |

Toolbar 按钮通过 `EditorMenuRegistry.ExecuteCommand("scene.play")` 调用。

## 已知限制（Phase 2）

| 问题 | 说明 |
|------|------|
| **Deferred Queue 精细分层** | Phase 1 的 `FlushDeferred(mask)` 全量 Flush。Phase 2 内部按标签分层队列。 |
| **Native 标签过滤** | C++ 侧尚未实现 `SceneSystemTags` 过滤，当前回退到无过滤版本。 |
| **Frame Step** | `StepFrame()` 接口已预留，`throw NotImplementedException`。 |
| **Game View Panel** | 独立游戏视口，Phase 2。 |
| **运行时修改回写** | 播放模式下的修改可选择性保留到编辑态，Phase 2。 |
| **快照清理** | 临时快照文件在 ExitPlay 时未删除（`VFS.Delete` 未实现），Phase 2。 |

## 命名空间冲突解决

`PlayMode` 目录命名空间 `Neverness.Editor.Scene.Private.PlayMode` 与类型 `Neverness.Editor.Core.Public.PlayMode` 冲突。通过 using alias 解决：

```csharp
using EditorPlayMode = Neverness.Editor.Core.Public.PlayMode;
```

## 验证方案

1. 编辑器打开场景 → 点击 Play → Gameplay + Physics + Always 系统开始 tick
2. 点击 Stop → 场景恢复到 Play 之前的状态（原子替换）
3. 播放期间修改实体属性 → 退出后属性恢复原值
4. 播放期间创建/删除实体 → 退出后实体恢复原状
5. 无场景时点击 Play → 按钮无响应（CanExecute 返回 false）
6. 暂停 → Always 系统继续 tick，Gameplay/Editor/Physics 暂停
7. 恢复 → Gameplay + Physics 继续 tick
8. 快照保存失败时 PlayMode 不切换（异常安全）
