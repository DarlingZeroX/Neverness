# 修复：TickActiveScene 未被调用导致 Viewport 无渲染

## 问题

`SceneManager.TickActiveScene(float)` 存在但从未被调用，导致：
- ECS System 不 Tick
- 场景数据不更新
- EditorViewport 无渲染内容

## 根因分析

### 调用链路（修复前）

```
Main.cpp 主循环
  → NNEngineRuntimeHost_TickManaged(dt)            [C++ ManagedBridge]
    → Entry.RuntimeTick(dt)                         [C# UnmanagedCallersOnly]
      → RuntimeMainLoop.Tick(dt)                    [Bootstrap]
        → RuntimeInitializer.Loop.Tick(ctx)         [RuntimeLoop]
          → SubsystemScheduler.TickBucket(Update)   [调度器]
            → ❌ SceneManager 不在桶中
```

### 断裂点

| 层 | 问题 |
|---|---|
| `SceneManager` | 纯业务类，未实现 `IManagedRuntimeSubsystem`，无法注册到调度器 |
| `RuntimeInitializer.RegisterDefaultSubsystems` | 空实现，无默认子系统 |
| `EditorApplicationRunner.Run()` | 创建了 SceneManager 但从未注册到 RuntimeLoop |

## 修复方案

### 架构

```
EditorApplicationRunner
  ├─ new SceneManager()              ← 已有
  ├─ new SceneSubsystem(sceneManager) ← 新增适配器
  └─ RuntimeInitializer.RegisterSubsystem(sceneSubsystem) ← 新增注册入口
        │
        ▼
      SubsystemScheduler.Register()
        → TickGroup.Update 桶
        → 每帧调 SceneSubsystem.Tick()
          → SceneManager.TickActiveScene(dt)
            → SceneWorld.Tick(dt)
```

### 修改文件

#### 1. `Runtime/Neverness.Runtime.Scene/Neverness.Runtime.Scene.csproj`

添加 `RuntimeLoop` 项目引用（`RuntimeLoop` 是纯 Runtime 程序集，无重依赖）。

```xml
<ProjectReference Include="..\Neverness.Runtime.RuntimeLoop\Neverness.Runtime.RuntimeLoop.csproj" />
```

#### 2. `Runtime/Neverness.Runtime.Scene/Public/SceneSubsystem.cs` （新建）

薄适配器，将 `SceneManager.TickActiveScene` 接入 `SubsystemScheduler` 调度。

```csharp
public sealed class SceneSubsystem : IManagedRuntimeSubsystem
{
    private readonly SceneManager _sceneManager;

    public RuntimeTickGroup TickGroup => RuntimeTickGroup.Update;

    public void Tick(in ManagedRuntimeFrameContext context)
    {
        _sceneManager.TickActiveScene(context.DeltaTimeSeconds);
    }
    // Initialize/Shutdown 为空 — SceneManager 由 Editor 生命周期管理
}
```

**设计原则**：
- `SceneManager` 保持纯净，不侵入接口实现
- `SceneSubsystem` 职责单一，仅做转发
- 放在 Scene 模块内部，内聚性好
- TickGroup = Update，与 Native SceneTick 对齐

#### 3. `Runtime/Neverness.Runtime.Bootstrap/RuntimeInitializer.cs`

添加公开的 `RegisterSubsystem` 方法，支持在 `Initialize()` 完成后补充注册子系统：

```csharp
public static void RegisterSubsystem(IManagedRuntimeSubsystem subsystem)
{
    ArgumentNullException.ThrowIfNull(subsystem);
    Loop?.Register(subsystem);
    if (s_initialized)
    {
        subsystem.Initialize(); // 迟到注册需补调 Initialize
    }
}
```

**关键**：`Initialize()` 内部先调 `RegisterDefaultSubsystems` 再调 `InitializeRegistered`。Editor 的 SceneSubsystem 在 `Initialize()` 完成之后才创建（SceneManager 在主循环第一帧实例化），所以 `RegisterSubsystem` 必须对迟到子系统补调 `Initialize()`。

#### 4. `Editor/NevernessEditor/EditorApplicationRunner.cs`

在模块安装块中注册 SceneSubsystem：

```csharp
SceneModule.Install(sceneManager);

/* 注册场景子系统到 RuntimeLoop，驱动 ECS Tick */
RuntimeInitializer.RegisterSubsystem(new SceneSubsystem(sceneManager));
```

## 修复后调用链路

```
Main.cpp → TickManaged(dt)
  → Entry.RuntimeTick(dt)
    → RuntimeMainLoop.Tick(dt)
      → RuntimeLoop.Tick(ctx)
        → SubsystemScheduler.TickBucket(Update, ctx)
          → SceneSubsystem.Tick(ctx)
            → SceneManager.TickActiveScene(dt)
              → SceneWorld.Tick(dt)
                → SceneSystemScheduler.Tick(Update, dt)
                → SceneNativeBridge.TickSystems(handle, dt)
                → SceneSystemScheduler.Tick(LateUpdate, dt)
                → SceneSystemScheduler.Tick(Render, dt)
                → Events.FlushDeferred()
```

## 后续关注

- **Runtime.Host Entry.cs**：独立运行时（非 Editor）路径同样未注册 SceneSubsystem。若需要独立运行时也驱动场景 Tick，应在 `RuntimeInitializer.RegisterDefaultSubsystems` 或独立的 Game 启动器中注册。
- **FixedUpdate**：当前 `SceneSubsystem` 仅 Tick Update 阶段。若需要 FixedUpdate 驱动物理等系统，可扩展为同时注册一个 `RuntimeTickGroup.FixedUpdate` 子系统调用 `SceneWorld.FixedTick`。
